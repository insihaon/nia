# Docker에서 로그 확인 방법

## 📋 CollectSvc.java 893-895 라인 로그 확인

### 추가된 로그:
```java
log.warn("NodeFactor 수집 실패 감지 - Node: {}, FatalErrors: {}, ErrCount: {}",
    ncc.getNodename(), ncc.getFatalErrors(), ncc.getErrCount());
```

### Docker에서 확인하는 명령어:

```bash
# 1. 해당 로그만 확인
docker logs ipsdn_opt_1 2>&1 | grep -i "NodeFactor 수집 실패 감지"

# 2. 최근 100줄에서 확인
docker logs ipsdn_opt_1 2>&1 | tail -100 | grep -i "NodeFactor 수집 실패 감지"

# 3. 최근 500줄에서 확인 (더 많은 로그)
docker logs ipsdn_opt_1 2>&1 | tail -500 | grep -i "NodeFactor 수집 실패 감지"

# 4. 실시간 모니터링
docker logs -f ipsdn_opt_1 2>&1 | grep -i "NodeFactor 수집 실패 감지"

# 5. 전체 컨텍스트와 함께 확인 (앞뒤 5줄)
docker logs ipsdn_opt_1 2>&1 | grep -B 5 -A 5 "NodeFactor 수집 실패 감지"

# 6. 날짜별로 확인 (예: 오늘)
docker logs ipsdn_opt_1 2>&1 --since "2025-01-01" | grep -i "NodeFactor 수집 실패 감지"
```

## 🔍 이 로그가 언제 출력되는가?

### 호출 경로:
```
Schedular.calculateDayDataAndOptimizeRoute() (매일 1시)
  ↓
collectSvc.checkCollecting(optDate)
  ↓
checkNodeFactorCollecting(measuredDate)
  ↓
makeMailContentForNodeFactor(nodeCollectCheckList, measuredDate)
  ↓
log.warn("NodeFactor 수집 실패 감지...") ← 여기서 출력
```

### 출력 조건:
- `ncc.getFatalErrors() >= 3` 일 때만 출력
- 즉, **연속으로 3개 이상의 nodefactor 데이터가 수집되지 않았을 때**
