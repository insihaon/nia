## 보류된 기능: 탭별 데이터 없을 때 메시지 표시

### 목표

Span Loss / Node Gain 등 데이터가 없는 탭 선택 시, 토폴로지를 숨기고 "선택된 지표의 데이터가 존재하지 않습니다." 메시지를 표시하려 했으나 실패.

### 시도 1: v-if / v-else (실패)

```vue
<div v-if="!topologyEmptyMessage">
  <Comp2DTopology ... />
</div>
<div v-if="topologyEmptyMessage" class="topology-empty-overlay">
  <p>{{ topologyEmptyMessage }}</p>
</div>
```

**실패 원인**: `v-if`로 Comp2DTopology가 DOM에서 제거되면, Map2D 라이브러리의 `initMap()`이 SVG 컨테이너를 찾지 못해 `translate(NaN,NaN)` 에러 발생. 재마운트 시 타이밍 문제로 정상 초기화 불가.

### 시도 2: v-show (실패)

```vue
<Comp2DTopology v-show="!topologyEmptyMessage" ... />
<div v-show="topologyEmptyMessage" class="topology-empty-overlay">
  <p>{{ topologyEmptyMessage }}</p>
</div>
```

**실패 원인**: `v-show`로 토폴로지를 `display:none` 처리해도 화면에 변화가 없었음. 원인 불확실 — Map2D 라이브러리가 SVG를 직접 조작하므로 Vue의 `v-show`가 내부 렌더링에 영향을 주지 못하거나, computed 반응성이 탭 전환과 맞물려 예상대로 동작하지 않았을 가능성.

### 시도 3: 오버레이 방식 (부분 성공)

```vue
<div style="position: relative;">
  <Comp2DTopology ... />  <!-- 항상 렌더링 -->
  <div v-if="topologyEmptyMessage" class="topology-empty-overlay">
    <p>{{ topologyEmptyMessage }}</p>
  </div>
</div>
```

**결과**: 전표 미선택 시 오버레이는 정상 동작. 그러나 탭별 데이터 유무에 따른 오버레이 전환은 동작하지 않음.

### 근본 원인 분석

1. **Map2D 라이브러리가 외부 minified 라이브러리**라 내부 상태를 Vue에서 제어하기 어려움
2. d3.js가 SVG를 직접 조작하므로 **Vue 반응성 시스템과 독립적**으로 동작
3. 탭 전환 시 d3의 `.text(undefined)` → 이전 텍스트 유지 동작이 근본 문제
4. `topologyNodes` 데이터 저장은 정상이었으나, 오버레이 전환이 Vue/Map2D 간 충돌로 동작하지 않음

### 향후 해결 방안

1. **Map2D 라이브러리 소스 수정**: `updateNodeInfoDisplayField` 내부에서 값이 없을 때 빈 문자열로 표시하도록 수정 (가장 근본적)
2. **라이브러리에 콜백 추가**: 탭 전환 후 표시된 값을 콜백으로 받아 Vue에서 판단
3. **SVG DOM 직접 조작**: `this.$refs.topology2d.$el.querySelector('svg')` 접근으로 d3 외부에서 visibility 제어

## 현재 최종 상태

- 전표 미선택 시 → 반투명 오버레이 + 안내 메시지 (동작)
- 전표 선택 + Rx/Tx 탭 → 토폴로지 정상 표시 (동작)
- 전표 선택 + Span Loss 탭 → **Rx/Tx 값이 그대로 남아있음** (미해결)
- 전표 선택 + NTD 탭 → NTD 값 정상 표시 (동작, 탭 전환 에러 수정됨)
