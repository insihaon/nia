## 4. 대응 방안 제안

### 단기 (UI 정리)

- `influencecircuit_count`, `equip_cnt`: Vue에서 해당 영역 숨김 처리 또는 0 표시 유지
- `nature_restoration`: 자연복구 색상 분기 제거, status 기반으로만 색상 결정
- 영향회선 탭: 데이터 없음 안내 문구 추가 또는 탭 숨김
- TT일괄처리 버튼: 비활성(disabled) 처리

### 중기 (NIA 테이블로 대체 - 실데이터 검증 완료)

topas 스키마 대신 NIA 자체 테이블로 대체 가능한 항목 확인됨:
- `equip_type`: `nia.tb_topology.equip_type` 사용 (ROADM/POTN 값 확인됨)
- `metype` → `model`: `nia.tb_equip_mst.model` 사용 (JOIN: topology.sysname = equip_mst.tid)
  - 단국장치: Utrans-6300p-V9, 중계기: Utrans-6300p-H3, OPN: OPN-3000
- 장비 한글명: `nia.tb_equip_mst.sysname` (예: 서울NIA, 대전ETRI, 중계_천안중_0204B)
- `officename`: 대체 불가 확정 (NIA에 국사 정보 없음)
- `rsspuslot`: 대체 불가 확정 (NIA ptpname 포맷이 TOPAS와 완전히 다름)

### 장기 (기능 재설계)

- `monitored_object` 재생성: `tid || '-' || ptpname || '-' || in_out` 포맷으로 대체 가능한지 기존 Vue 파싱 로직(L275~286)과 대조 필요
- 영향회선 기능: `mba.tb_ticket_ic_mst` 테이블 재생성 또는 대체 데이터 소스 확보
- TT일괄처리: `rca.tb_rca_ticket_topas_tt_current` 테이블 재생성 필요
- 고장원인/책임분류: 코드 테이블(`rca.code_reason`, `rca.code_facility`) 재생성 필요
