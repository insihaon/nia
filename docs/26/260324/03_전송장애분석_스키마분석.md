## 1. 스키마 개요

### 1.1 매퍼에서 활성 사용 중인 스키마

| 스키마 | 용도 |
|--------|------|
| **rca** | RCA(Root Cause Analysis) 엔진 - 전표, 알람 분석, 처리 상태 |
| **mba** | MBA(Momentary Breakoff Analysis) - 순단장애, 예지보전, 광성능 |

| **aib_inst** | 설치 위치 정보 (instlocation) | 주석 처리됨 |
| **inspector** | CIT2 검사 결과 | 주석 처리됨 |
| **post** | 전송회선 포스트 정보 | 주석 처리됨 |

## 6. 구체적 보완 제안

### 6.1 즉시 적용 가능 (기존 쿼리 개선)

#### (1) SELECT_TICKET_CUR_LIST에 `construction` 플래그 추가

```sql
-- 현재: 공사 여부를 확인할 수 없음
-- 제안: T.construction 컬럼 추가
, T.CONSTRUCTION AS CONSTRUCTION
```

**효과**: 공사 중 발생 장애를 UI에서 구분 표시 가능

#### (2) SELECT_TICKET_CUR_LIST에 `trunk_id` 추가

```sql
-- 현재: trunk_name만 조회
-- 제안: trunk_id도 함께 조회하여 MBA 토폴로지 연계 강화
, T.TRUNK_ID AS TRUNK_ID
```

**효과**: MBA 테이블과 JOIN시 trunk_id를 키로 사용 가능 (trunk_name보다 안정적)

#### (3) SELECT_MBA_LOW_ALARM_LIST 컬럼 확장

```sql
-- 현재: sysname, value_max, value_cur, value_min, is_low_signal, diff만 조회
-- 제안: 토폴로지 연계 및 상세 정보 추가
, T.trunk_id AS trunk_id
, T.trunk_name AS trunk_name
, T.ptpname AS ptpname
, T.in_out AS in_out
, T.direction AS direction
, T.routenum AS routenum
, TO_CHAR(T.ocrtime, 'YYYY-MM-DD HH24:MI:SS') AS ocrtime
```

**효과**: 저광레벨 알람의 정확한 위치(어떤 트렁크, 어떤 포트, IN/OUT 방향) 파악 가능

#### (4) SELECT_MBA_TOPOLOGY_LIST 컬럼 확장

```sql
-- 현재: TRUNK_NAME, SYSNAME, ROUTENUM만 조회
-- 제안:
, T.TRUNK_ID AS TRUNK_ID
, T.UP_PTPNAME AS UP_PTPNAME
, T.DOWN_PTPNAME AS DOWN_PTPNAME
```

**효과**: 토폴로지 뷰에서 노드 간 연결 포트 정보 표시 가능

#### (5) 알람 레벨 정보 추가 (SELECT_TICKET_CUR_LIST)

```sql
-- AL 서브쿼리에 알람 레벨 추가
, AL.ROOT_CAUSE_ALARM_LEVELA AS ROOT_CAUSE_ALARM_LEVELA
, AL.ROOT_CAUSE_ALARM_LEVELZ AS ROOT_CAUSE_ALARM_LEVELZ
, AL.ROOT_CAUSE_PTPNAMEA AS ROOT_CAUSE_PTPNAMEA
, AL.ROOT_CAUSE_PTPNAMEZ AS ROOT_CAUSE_PTPNAMEZ
```

**효과**: 전표 목록에서 알람 심각도(Critical/Major/Minor) 즉시 확인 가능

### 6.2 신규 쿼리 추가 제안

#### (1) MBA 전용 전표 조회 (tb_mba_ticket_current 활용)

```sql
<select id="SELECT_MBA_TICKET_LIST" parameterType="hashmap" resultType="hashmap">
    SELECT
        ticket_id, cluster_no, ticket_type,
        TO_CHAR(ticket_generation_time, 'YYYY-MM-DD HH24:MI:SS') AS generation_time,
        TO_CHAR(fault_time, 'YYYY-MM-DD HH24:MI:SS') AS fault_time,
        root_cause_type, root_cause_code, root_cause_domain,
        status, direction, trunk_id, trunk_name,
        total_related_alarm_cnt, occur
    FROM mba.tb_mba_ticket_current
    WHERE parent_ticket_id IS NULL
    <if test="TICKET_ID != null"> AND ticket_id = #{TICKET_ID} </if>
    <if test="FROM != null"> AND ticket_generation_time >= #{FROM}::TIMESTAMP </if>
    <if test="TO != null"> AND ticket_generation_time &lt;= #{TO}::TIMESTAMP </if>
    ORDER BY ticket_generation_time DESC
</select>
```

**효과**: MBA 전표를 RCA 전표와 분리하여 조회 - 성능 개선 및 MBA 전용 필터 적용 가능

#### (2) MBA 케이블 정보 조회 (tb_mba_ticket_cable 활용)

```sql
<select id="SELECT_MBA_TICKET_CABLE_LIST" parameterType="hashmap" resultType="hashmap">
    SELECT
        ticket_id, ticket_cable_id,
        is_root_row_signal_line,
        tida, tidz,
        sysnamea, sysnamez,
        ptpnamea, ptpnamez
    FROM mba.tb_mba_ticket_cable
    WHERE ticket_id = #{TICKET_ID}
</select>
```

**효과**: 순단장애 케이블 양단 정보 조회

#### (3) 광 기준값 비교 조회 (tb_roadm_optical_reference 활용)

```sql
<select id="SELECT_MBA_OPTICAL_REFERENCE" parameterType="hashmap" resultType="hashmap">
    SELECT
        ref.trunk_id, ref.trunk_name,
        ref.sysname, ref.ptpname, ref.in_out,
        ref.reference_value,
        TO_CHAR(ref.renewal_date, 'YYYY-MM-DD') AS renewal_date,
        cur.value_cur,
        (cur.value_cur - ref.reference_value) AS deviation
    FROM mba.tb_roadm_optical_reference ref
    LEFT JOIN mba.tb_roadm_low_optical_performance cur
        ON ref.trunk_id = cur.trunk_id
        AND ref.sysname = cur.sysname
        AND ref.ptpname = cur.ptpname
        AND ref.in_out = cur.in_out
        AND cur.ticket_id = #{TICKET_ID}
    WHERE ref.trunk_name = #{TRUNK_NAME}
</select>
```

**효과**: 현재 광 수치와 기준값 편차를 한번에 조회 - 예지보전 판단 근거 제공
