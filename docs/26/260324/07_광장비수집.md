# NIA 광장비 정보 수집 체계

> 분석일: 2026-03-24
> 수집 모듈: `NiaEmsLinkageApplication` (TELNET/TL1 기반)

---

## 1. 수집 주체

광장비 정보 수집은 `NiaEmsLinkageApplication`의 `RoadmEmsMmcServiceImpl`이 담당한다.
EMS(Element Management System)에 TELNET으로 접속하여 TL1 MMC 명령어를 실행하고, 응답을 파싱하여 DB에 저장한다.

---

## 2. 수집 항목별 상세

### 2.1 장비 마스터 (System Info)

| 항목 | 값 |
|------|-----|
| **MMC 명령** | `RTRV-SYS:{TID}::123;` |
| **수집 메서드** | `updateSystemInfo()` |
| **저장 테이블** | `nia.tb_equip_mst` |
| **수집 트리거** | `getSystemList()`로 TID 목록 수신 후 각 TID별 실행 |

**`nia.tb_equip_mst` 컬럼:**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `tid` | varchar | 장비 TID (Terminal ID) |
| `sysname` | varchar | 시스템명 |
| `psu` | varchar | PSU 정보 |
| `model` | varchar | 장비 모델명 |
| `sid` | varchar | SID |
| `sw_ver` | varchar | 소프트웨어 버전 |
| `sw_chksum` | varchar | 소프트웨어 체크섬 |
| `sw_date` | varchar | 소프트웨어 날짜 |
| `sw_time` | varchar | 소프트웨어 시간 |

**수집 흐름:**
```
RTRV-NET:::; (전체 장비 목록 획득)
  → TID 리스트를 DataShareBean에 저장
  → 각 TID별 RTRV-SYS:{TID}::123; 실행
  → 응답 파싱 → tb_equip_mst_yd (임시) INSERT
  → tb_equip_mst UPDATE (equipmentMapper.updateEquipMst)
  → tb_equip_mst_yd 삭제
```

### 2.2 SIPC 정보 (서브시스템)

| 항목 | 값 |
|------|-----|
| **MMC 명령** | `RTRV-SIPC:{IP}-SH1::::;` |
| **수집 메서드** | `roadmSipcMMC()` |
| **저장 테이블** | `nia.tb_equip_sipc` |

**`nia.tb_equip_sipc` 컬럼:**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `tid` | varchar | 장비 TID |
| `system_name` | varchar | 서브시스템명 (예: SH1, SH2) |

**수집 흐름:**
```
tb_equip_mst에서 장비 IP 목록 조회
  → 각 장비별 RTRV-SIPC:{IP}-SH1::::; 실행
  → 응답 파싱 → tb_equip_sipc_tmp INSERT
  → tb_equip_sipc 갱신
```

### 2.7 광성능 데이터 (15분 주기)

| 항목 | 값 |
|------|-----|
| **MMC 명령** | `RTRV-PM:{SID}-{NAME}::::SIGNAL=AMPPWR,TYPE=15M,INTERVAL=CURR;` |
| **수집 메서드** | `roadmPmMMC()` |
| **저장 테이블** | `mba.tb_roadm_optical_performance_hist`, `mba.tb_roadm_low_optical_performance` |

**수집 흐름:**
```
tb_equip_sipc에서 SIPC 목록 조회
  → 각 SIPC별 RTRV-PM 명령 실행
  → 응답 파싱 (value_cur, value_max, value_min 추출)
  → mba.fc_set_performance_preprocessor() 호출 (전처리)
  → RabbitMQ Queue → AI 서버 전달
  → 성능 이력(hist), 저광레벨(low), 일별(daily) 테이블 적재
```

## 3. 수집 체계 요약

```
NiaEmsLinkageApplication
├── getSystemList()          → RTRV-NET        → TID 목록 획득
├── updateSystemInfo()       → RTRV-SYS        → nia.tb_equip_mst (장비 마스터)
├── roadmSipcMMC()           → RTRV-SIPC       → nia.tb_equip_sipc (서브시스템)
├── roadmNetWorkMmc()        → RTRV-NETWORK    → nia.tb_topology (NNI 토폴로지)
│                                               → nia.tb_equip_network
├── createRoadmUniTopology() → RTRV-OPC        → nia.tb_uni_topology (UNI 토폴로지)
│                                               → nia.tb_equip_opc
├── roadmSlotMMC()           → RTRV-SLOT       → nia.tb_equip_slot (슬롯)
├── roadmAlarmMMC()          → RTRV-ALM        → 경보 수집 (별도 처리)
└── roadmPmMMC()             → RTRV-PM         → mba.tb_roadm_optical_performance_hist
                                                → mba.tb_roadm_low_optical_performance
```
