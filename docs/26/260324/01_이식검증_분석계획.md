# Vue 화면 이식 검증 분석 보고서

## 배경

Oasis_untact 시스템의 두 화면을 NIA Web2024로 이식한 작업의 검증.
데이터 모델이 다르므로 SQL 일부는 주석처리/스텁(WHERE 1=0) 처리됨.

## 3. 공통 인프라

### API 차이 (nia.js vs rca.js)

- apiRcaRequest: 동일 시그니처, URL 패턴만 다름
- apiUserProcess: Source는 RabbitMQ 메시지 방식, Target은 SQL 직접 처리 방식
  - `USER_PROCESS_MBA`, `USER_PROCESS_EQUIP` SQL이 XML에 부재 -> **인지/마감 처리 에러**

### 공유 SQL 파일

transmissionFaultAnalytics.xml이 TransientOutage와 PredictiveMaintenance 모두의 쿼리를 포함.
총 쿼리 18개 중 활성 10개, 스텁 8개.

## 5. 검증 방법

### 빌드 검증
```bash
cd apps/Web2024/FE && npm run build
```
- 빌드 에러 없는지 확인 (import 경로 오류, 미등록 컴포넌트 등)

### 화면별 검증

**TransientOutageList:**
- /transientOutage/transientOutageList 접근 -> 전표 목록 조회
- 전표 더블클릭 -> 토폴로지 표시
- 인지/마감 버튼 -> 모달 열림 및 처리
- 일괄처리 버튼 -> 모달 열림

**PredictiveMaintenanceTicket:**
- /transientOutage/predictiveMaintenance 접근 -> 전표 목록 조회
- 전표 클릭 -> 토폴로지 표시 (Map2D 구현 후)
- 검토의견 모달 -> CRUD 작동
- 토폴로지 노드 클릭 -> 차트 팝업 (pmmChart 이식 후)

### SQL 검증
- 스텁 쿼리의 원본 테이블이 NIA DB에 존재하는지 DBA 확인 필요
- 주석 처리된 컬럼의 원본 테이블/컬럼 존재 여부 확인
