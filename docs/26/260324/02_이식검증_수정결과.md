## 수정 완료 항목

### Phase 1: P0 런타임 에러 수정 (3건)

#### 1. Store pmmTicketList 누락 보강

- **파일**: `apps/Web2024/FE/src/store/modules/untact.js`
- **문제**: PredictiveMaintenanceTicket.vue에서 `this.$store.dispatch('untact/insertTicketPmmList', ...)` 호출 시 action이 없어 런타임 에러 발생
- **수정 내용**:
  - `state.pmmTicketList` 추가 (키: `ticketno`, MBA는 `ticket_id`)
  - `getPmmTicketList()` 헬퍼 함수 추가
  - mutation: `UPSERT_TICKET_PMM_LIST`, `DELETE_TICKET_PMM_LIST`
  - action: `insertTicketPmmList`, `upsertTicketPmmList`, `deleteTicketPmmList`
  - getter: `pmmTicketList`

#### 2. CSS 선택자 불일치 수정

- **파일**: `apps/Web2024/FE/src/views-nia/pages/PredictiveMaintenance/PredictiveMaintenanceTicket.vue` (line 407)
- **문제**: 글로벌 SCSS가 `.PmmTicket` 선택자를 사용하지만 routeName이 `PredictiveMaintenanceTicket`으로 변경되어 스타일 전체 미적용
- **수정 내용**: `.PmmTicket` → `.PredictiveMaintenanceTicket`

#### 3. loadAzList API 응답 키 미변경 수정

- **파일**: `apps/Web2024/FE/src/views-nia/pages/PredictiveMaintenance/PredictiveMaintenanceTicket.vue` (line 246)
- **문제**: NIA API는 `res.result`로 응답하지만 `result?.data`로 접근하여 토폴로지 노드 데이터가 항상 빈 배열
- **수정 내용**: `result?.data || []` → `result?.result || []`

### Phase 2: P1 핵심 기능 수정 (2건)

#### 4. 토폴로지 Map2D 전체 구현 (스텁 → 완전 구현)

- **파일**: `apps/Web2024/FE/src/views-nia/pages/PredictiveMaintenance/component/Map2D.vue`
- **문제**: 28줄 스텁 상태로 D3 토폴로지 로직 전혀 없음. 우측 토폴로지 패널 완전 미작동
- **수정 내용**: TransientOutage의 `topology2d/index.vue`를 기반으로 PMM용 전체 포팅
  - D3 라이브러리 로드 (`map2d_untact.min.js`)
  - `Base` mixin 확장, `apiRcaRequest` 연동
  - PMM 키: `ticket.ticketno` (MBA는 `ticket.ticket_id`)
  - PMM 파라미터: `TICKET_TYPE: 'PM'` (MBA는 `'RT'`)
  - `nodeInfoOption` prop 추가 (부모 컴포넌트에서 전달)
  - SVG ID: `pmm_topology_container` (MBA와 충돌 방지)
  - CSS 클래스: `.PmmTopology2D` (MBA는 `.MbaTopology2D`)
  - `loadList`, `selectedTopologyItem` 이벤트 emit 동일

#### 5. monitored_object NPE 방어 코드 추가

- **파일**: `apps/Web2024/FE/src/views-nia/pages/TransientOutage/TransientOutageList.vue` (line 276)
- **문제**: SQL에서 `monitored_object` 컬럼이 주석 처리되어 미반환. `row.monitored_object.substr()` 호출 시 undefined 에러
- **수정 내용**: `if (!monitoredInfo) return false` null 체크 추가

#### 7. USER_PROCESS_MBA SQL 추가

- **파일**: `apps/Web2024/BE/app-nia/src/main/resources/mapper/db2nd/transmissionFaultAnalytics.xml`
- **문제**: `apiUserProcess`가 SQL ID `USER_PROCESS_MBA`를 호출하지만 XML에 정의 안됨. 인지/마감 버튼 클릭 시 서버 에러
- **선행조건**: NIA의 티켓 처리 테이블 구조 확인 후 UPDATE 쿼리 작성 필요
- **영향**: 순단장애 전표 인지/마감 처리 불가
