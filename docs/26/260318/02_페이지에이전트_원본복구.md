# NIA Page-Agent 활용 솔루션 가이드

> NIA 네트워크 관제 플랫폼의 AI 어시스턴트(Page-Agent)를 통해 운용자가 자연어로 수행할 수 있는 업무 시나리오

---

## 1. 즉시 활용 가능한 솔루션 (현재 기능으로 구현 가능)

### 1.1 멀티스텝 장애 대응 자동화

운용자가 알람을 확인하고 대응하는 과정을 한 문장으로 자동화합니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "현재 미처리 알람 중 가장 오래된 것부터 인지 처리해줘" | 대시보드 IP/전송망 알람 테이블 → 상태 필터링 → 시간순 정렬 → '인지' 버튼 클릭 |
| 2 | "이 알람 인지하고 바로 조치요청까지 진행해줘" | '인지' 버튼 클릭 → 조치요청 팝업 열기 → 양식 자동 채우기 |
| 3 | "전송망 알람 전체 일괄 마감 처리해줘" | 전송망 테이블 전체 선택 → TT일괄처리 버튼 클릭 → 마감 처리 |

**활용 가치**: 반복적인 클릭 작업 5~10단계를 한 문장으로 단축. 야간 당직 시 빠른 대응 가능.

---

### 1.2 조건부 화면 이동 + 필터 자동 설정

단순 화면 이동이 아닌, 특정 조건의 데이터를 바로 조회하는 시나리오입니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "어제 발생한 순단장애 중 미마감 건만 보여줘" | 장애감시 → 순단장애 메뉴 이동 → 날짜 필터(어제) → 상태 필터(미마감) 설정 |
| 2 | "이용기관별 트래픽 Top 10 조회해줘" | 자원성능감시 → 이용기관별 트래픽 분석 이동 → Top N 설정 → 조회 실행 |
| 3 | "장비 사용량에서 CPU 80% 이상인 장비만 필터해줘" | 장비사용량 화면 이동 → CPU 필터 80% 이상 설정 → 조회 |
| 4 | "서울 노드 리스트만 조회해줘" | 자원구성관리 → 노드 리스트 → 검색 필터에 '서울' 입력 → 조회 |

**활용 가치**: 메뉴 찾기 + 필터 설정을 한번에. 신규 운용자도 즉시 원하는 데이터에 접근 가능.

---

### 1.3 데이터 내보내기 자동화

보고서 작성을 위한 데이터 추출을 자연어로 수행합니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "이번 주 SOP 이력 엑셀로 다운로드해줘" | 이력관리 → SOP 이력조회 → 날짜 필터(이번 주) → 조회 → 엑셀 내보내기 |
| 2 | "장애이력 최근 30일치 뽑아줘" | 장애이력 화면 → 30일 기간 설정 → 조회 → 엑셀 내보내기 |
| 3 | "Syslog 이력에서 오늘 발생한 것만 엑셀로 저장" | Syslog 이력조회 → 오늘 날짜 → 조회 → 내보내기 |

**활용 가치**: 주간/월간 보고서용 데이터 추출 시간을 분 단위에서 초 단위로 단축.

---

### 1.4 예지보전 워크플로우

광모듈 장애 예측 티켓 처리 흐름을 자동화합니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "예지보전 미처리 티켓 중 편차가 큰 순서로 정렬해줘" | 예지보전 화면 이동 → Node Total Deviation 기준 내림차순 정렬 |
| 2 | "이 예지보전 티켓의 영향 구간 토폴로지 보여줘" | 해당 티켓 선택 → 토폴로지 2D 맵 열기 |
| 3 | "원홉 캐리어 제외하고 보여줘" | '원홉 캐리어 제외' 체크박스 클릭 → 필터 적용 |

**활용 가치**: 예지보전 분석 시 필터/정렬/시각화 단계를 음성이나 텍스트로 빠르게 전환.

---

### 1.5 운용 현황 모니터링 전환

여러 모니터링 화면을 빠르게 오가며 현황을 파악합니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "AI 모니터링 화면으로 전환해줘" | 운용현황 → AI 모니터링 이동 |
| 2 | "서버 모니터링에서 이상 있는 서버만 보여줘" | 서버 모니터링 이동 → 이상 상태 필터 |
| 3 | "자가처리 현황 대시보드를 일별로 보여줘" | 자가처리 현황 → 일별 통계 뷰 전환 |
| 4 | "운용 현황 통계를 월별로 바꿔줘" | 운용 현황 통계 → 월별 차트 전환 |

## 2. 고급 활용 솔루션 (추가 개발 권장)

### 2.1 크로스 화면 요약 브리핑

**시나리오**: "현재 상황 브리핑해줘"

Page-Agent가 여러 화면을 순회하며 핵심 정보를 수집하고 요약합니다.

```
동작 흐름:
1. 대시보드 → 미처리 알람 건수 확인 (IP: N건, 전송: N건)
2. 순단장애 → 미마감 건수 확인
3. 예지보전 → 미처리 예측 티켓 수 확인
4. 하단바 → 시스템 상태(AI연동, 수집/전송) 확인
5. 결과를 자연어로 요약하여 채팅창에 표시
```

**예상 결과 메시지**:
> 현재 미처리 IP 알람 3건, 전송망 알람 1건이 있습니다.
> 순단장애 미마감 2건, 예지보전 미처리 티켓 5건입니다.
> AI연동 시스템은 정상이며, 수집/전송 시스템도 정상입니다.

**필요 개발**: Page-Agent `customTools`에 '브리핑 수집' 도구 추가. 각 화면의 DOM에서 핵심 수치를 추출하는 로직 구현.

---

### 2.2 자연어 기반 비교 분석

**시나리오**: "어제와 오늘 자가처리 현황 비교해줘"

```
동작 흐름:
1. 자가처리 현황 → 어제 날짜로 통계 조회 → 수치 기록
2. 오늘 날짜로 전환 → 수치 기록
3. 두 수치를 비교하여 증감 분석 결과 표시
```

**예상 결과 메시지**:
> 어제 대비 오늘 자가처리 건수가 12% 증가했습니다 (어제 45건 → 오늘 51건).
> 자가처리 성공률은 89%에서 92%로 개선되었습니다.

**필요 개발**: `getPageInstructions()` 콜백에서 화면별 데이터 추출 지침 제공. Page-Agent의 `onAfterStep` 훅에서 수집 데이터를 임시 저장.

---

### 2.3 연속 작업 매크로 (Workflow Chain)

**시나리오**: "전송망 알람 전체 확인하고 인지 처리 후 SOP 이력도 같이 조회해줘"

```
동작 흐름:
1. 대시보드 전송망 알람 테이블 확인
2. 미인지 알람 선택 → 인지 처리
3. SOP 이력조회 메뉴 이동
4. 방금 처리한 알람 관련 SOP 이력 필터링
5. 결과 보고
```

**필요 개발**: 작업 간 컨텍스트(알람 ID, 장비명 등)를 유지하는 세션 변수 관리. `customTools`에 '작업 체이닝' 도구 추가.

---

### 2.4 즐겨찾기 화면 세트 (Quick View)

**시나리오**: "아침 점검 루틴 실행해줘"

사전 등록된 화면 세트를 순서대로 열어 보여줍니다.

```
아침 점검 루틴 예시:
1. 대시보드 전체 현황 확인 (3초 대기)
2. 순단장애 미마감 건 확인
3. 예지보전 신규 티켓 확인
4. 서버 모니터링 이상 여부 확인
5. "아침 점검 완료. 이상 N건 발견" 보고
```

**필요 개발**: 루틴(매크로) 등록/관리 UI. localStorage 기반 루틴 저장. Page-Agent `systemInstructions`에 루틴 목록 주입.

---

### 2.5 알람 상관 분석 안내

**시나리오**: "이 알람과 관련된 다른 알람이나 장애가 있어?"

```
동작 흐름:
1. 현재 보고 있는 알람의 장비명/IP 추출
2. 순단장애 화면에서 같은 장비 관련 건 검색
3. Syslog에서 동일 장비 이력 검색
4. 관련 건이 있으면 연관 정보 요약 표시
```

**필요 개발**: Page-Agent `customTools`에 '상관분석' 도구 추가. 장비/IP 기반 크로스 검색 로직.

## 4. 추가 개발 제안

### 4.1 `systemInstructions` 설정 (우선순위: 높음)

Page-Agent 초기화 시 NIA 도메인 지식을 주입합니다. 현재 코드에는 이 설정이 없습니다.

```javascript
this.pageAgent = new PageAgent({
  model: 'gemini-2.5-flash',
  baseURL: '/gemini-api',
  apiKey: process.env.VUE_APP_GEMINI_API_KEY,  // 환경변수로 이동
  language: 'ko-KR',
  systemInstructions: `
    당신은 NIA 네트워크 관제 플랫폼의 AI 어시스턴트입니다.
    사용자는 네트워크 운용자이며, 알람 관리, 장애 대응, 성능 모니터링을 수행합니다.

    화면 이동 시 좌측 사이드바 메뉴를 클릭하여 이동하세요.
    데이터 조회 시 필터 조건을 먼저 설정한 후 '조회' 버튼을 클릭하세요.
    날짜 관련 요청 시 datepicker를 사용하세요.

    주요 용어:
    - 순단장애: 짧은 시간 동안 발생하는 네트워크 단절 장애
    - 예지보전: 광모듈의 장애를 사전에 예측하는 기능
    - SOP: 표준운영절차 (Standard Operating Procedure)
    - 인지: 알람을 확인했음을 표시하는 처리
    - 마감: 장애 처리를 완료하는 행위
    - TT: Trouble Ticket (장애 티켓)
  `,
})
```

### 4.2 `getPageInstructions` 콜백 (우선순위: 높음)

화면별 맞춤 지침을 제공하여 Page-Agent의 정확도를 높입니다.

```javascript
this.pageAgent = new PageAgent({
  // ...기존 설정
  getPageInstructions: () => {
    const path = this.$route.path
    const instructions = {
      '/dashBoard/index': `
        이 화면은 메인 대시보드입니다.
        상단에 AI제어 시스템 처리량 차트가 있습니다.
        중앙에 IP-SDN 네트워크와 전송망 알람 테이블이 있습니다.
        알람 테이블에서 행을 클릭하면 상세 정보를 볼 수 있습니다.
        '인지' 버튼으로 알람을 인지 처리할 수 있습니다.
      `,
      '/transientOutage/transientOutageList': `
        이 화면은 순단장애 목록입니다.
        각 티켓 카드에 상태(INIT/ACK/FIN), 케이블 정보, 영향 건수가 표시됩니다.
        '인지' 버튼으로 확인, '마감' 버튼으로 종료 처리합니다.
        'TT일괄처리' 버튼으로 여러 건을 한번에 처리할 수 있습니다.
      `,
      '/transientOutage/predictiveMaintenance': `
        이 화면은 예지보전 티켓 목록입니다.
        Node Total Deviation(dB) 값이 클수록 장애 가능성이 높습니다.
        '원홉 캐리어 제외' 체크박스로 단순 구간을 제외할 수 있습니다.
        티켓을 클릭하면 영향 구간과 토폴로지를 확인할 수 있습니다.
      `,
    }
    return instructions[path] || ''
  },
})
```

### 4.3 `customTools` 확장 (우선순위: 중간)

NIA 전용 도구를 추가하여 Page-Agent의 기능을 확장합니다.

```javascript
this.pageAgent = new PageAgent({
  // ...기존 설정
  customTools: [
    {
      name: 'get_alarm_summary',
      description: '현재 대시보드의 알람 현황 요약을 반환합니다',
      parameters: {},
      func: async () => {
        // Vuex store에서 직접 알람 데이터 조회
        const ipAlarms = this.$store.state.nia.ipAlarmCount
        const transAlarms = this.$store.state.nia.transAlarmCount
        return `IP 알람: ${ipAlarms}건, 전송망 알람: ${transAlarms}건`
      }
    },
    {
      name: 'navigate_to',
      description: '지정된 NIA 화면으로 이동합니다',
      parameters: { screen_name: { type: 'string', description: '화면 한글명' } },
      func: async ({ screen_name }) => {
        const routeMap = {
          '대시보드': '/dashBoard/index',
          '순단장애': '/transientOutage/transientOutageList',
          '예지보전': '/transientOutage/predictiveMaintenance',
          'AI 모니터링': '/operationStatusScreen/aiMonitoring',
          // ... 전체 화면 맵
        }
        const path = routeMap[screen_name]
        if (path) {
          this.$router.push(path)
          return `${screen_name} 화면으로 이동했습니다`
        }
        return `'${screen_name}' 화면을 찾을 수 없습니다`
      }
    },
    {
      name: 'export_current_data',
      description: '현재 화면의 데이터를 엑셀로 내보냅니다',
      parameters: {},
      func: async () => {
        // 현재 화면의 exportExcel 함수 호출
        const exportBtn = document.querySelector('[data-action="export"]')
        if (exportBtn) { exportBtn.click(); return '엑셀 내보내기 완료' }
        return '이 화면에서는 내보내기를 지원하지 않습니다'
      }
    },
  ],
})
```

### 4.5 채팅 연동 (우선순위: 중간)

현재 Page-Agent와 기존 챗봇이 분리되어 있습니다. 사용자가 챗봇 입력창에서 바로 Page-Agent 명령을 실행할 수 있도록 연동합니다.

```javascript
// 챗봇 메시지 전송 시 Page-Agent 명령 감지
async sendMessage(message) {
  if (message.startsWith('/do ') || message.startsWith('해줘')) {
    // Page-Agent에 작업 위임
    await this.pageAgent.run(message)
    return
  }
  // 기존 챗봇 로직 유지
  this.existingChatLogic(message)
}
```
