# NIA Page-Agent 활용 솔루션 가이드

> NIA 네트워크 관제 플랫폼의 AI 어시스턴트(Page-Agent)를 통해 운용자가 자연어로 수행할 수 있는 업무 시나리오
>
> **주의**: PageAgent는 텍스트 기반 DOM 조작 방식으로 동작합니다. `el-date-picker`의 날짜 변경은 현재 정상 동작하지 않습니다. 날짜 관련 시나리오는 [날짜 문제 해결 Plan](#날짜-문제-해결-plan)을 참고하세요.

---

## 1. 즉시 활용 가능한 솔루션 (현재 기능으로 구현 가능)

### 1.1 멀티스텝 장애 대응 자동화

운용자가 알람을 확인하고 대응하는 과정을 한 문장으로 자동화합니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "현재 미처리 알람 중 가장 오래된 것부터 인지 처리해줘" | 대시보드 IP/전송망 알람 테이블 → 상태 필터링 → 시간순 정렬 → '인지' 버튼 클릭 |
| 2 | "이 알람 인지하고 바로 조치요청까지 진행해줘" | '인지' 버튼 클릭 → 조치요청 팝업 열기 → 양식 자동 채우기 |
| 3 | "전송망 알람 전체 일괄 마감 처리해줘" | 전송망 테이블 전체 선택 → TT일괄처리 버튼 클릭 → 마감 처리 |

**활용 가치**: 반복적인 클릭 작업 5~10단계를 한 문장으로 단축. 야간 당직 시 빠른 대응 가능.

---

### 1.2 조건부 화면 이동 + 필터 자동 설정

단순 화면 이동이 아닌, 특정 조건의 데이터를 바로 조회하는 시나리오입니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "이용기관별 트래픽 Top 10 조회해줘" | 자원성능감시 → 이용기관별 트래픽 분석 이동 → Top N 설정 → 조회 실행 |
| 2 | "장비 사용량에서 CPU 80% 이상인 장비만 필터해줘" | 장비사용량 화면 이동 → CPU 필터 80% 이상 설정 → 조회 |
| 3 | "서울 노드 리스트만 조회해줘" | 자원구성관리 → 노드 리스트 → 검색 필터에 '서울' 입력 → 조회 |
| 4 | "순단장애 중 미마감 건만 보여줘" | 장애감시 → 순단장애 메뉴 이동 → 상태 필터(미마감) 설정 |
| 5 | "경보 목록에서 상태를 '미처리'로 필터링해줘" | 알람 테이블 → el-select 상태 필터 → '미처리' 선택 |

**활용 가치**: 메뉴 찾기 + 필터 설정을 한번에. 신규 운용자도 즉시 원하는 데이터에 접근 가능.

### 1.3 예지보전 워크플로우

광모듈 장애 예측 티켓 처리 흐름을 자동화합니다.

| # | 사용자 명령 예시 | Page-Agent 동작 |
|---|----------------|----------------|
| 1 | "예지보전 미처리 티켓 중 편차가 큰 순서로 정렬해줘" | 예지보전 화면 이동 → Node Total Deviation 기준 내림차순 정렬 |
| 2 | "이 예지보전 티켓의 영향 구간 토폴로지 보여줘" | 해당 티켓 선택 → 토폴로지 2D 맵 열기 |
| 3 | "원홉 캐리어 제외하고 보여줘" | '원홉 캐리어 제외' 체크박스 클릭 → 필터 적용 |
| 4 | "첫 번째 예지보전 티켓의 리뷰 의견을 열어줘" | 테이블 행 클릭 → 리뷰 의견 모달 열기 |

**활용 가치**: 예지보전 분석 시 필터/정렬/시각화 단계를 음성이나 텍스트로 빠르게 전환.

## 2. 고급 활용 솔루션 (추가 개발 권장)

### 2.1 크로스 화면 요약 브리핑

**시나리오**: "현재 상황 브리핑해줘"

Page-Agent가 여러 화면을 순회하며 핵심 정보를 수집하고 요약합니다.

```
동작 흐름:
1. 대시보드 → 미처리 알람 건수 확인 (IP: N건, 전송: N건)
2. 순단장애 → 미마감 건수 확인
3. 예지보전 → 미처리 예측 티켓 수 확인
4. 하단바 → 시스템 상태(AI연동, 수집/전송) 확인
5. 결과를 자연어로 요약하여 채팅창에 표시
```

**예상 결과 메시지**:
> 현재 미처리 IP 알람 3건, 전송망 알람 1건이 있습니다.
> 순단장애 미마감 2건, 예지보전 미처리 티켓 5건입니다.
> AI연동 시스템은 정상이며, 수집/전송 시스템도 정상입니다.

**필요 개발**: Page-Agent `customTools`에 '브리핑 수집' 도구 추가. 각 화면의 DOM에서 핵심 수치를 추출하는 로직 구현.

---

### 2.2 연속 작업 매크로 (Workflow Chain)

**시나리오**: "전송망 알람 전체 확인하고 인지 처리 후 SOP 이력도 같이 조회해줘"

```
동작 흐름:
1. 대시보드 전송망 알람 테이블 확인
2. 미인지 알람 선택 → 인지 처리
3. SOP 이력조회 메뉴 이동
4. 방금 처리한 알람 관련 SOP 이력 필터링
5. 결과 보고
```

**필요 개발**: 작업 간 컨텍스트(알람 ID, 장비명 등)를 유지하는 세션 변수 관리. `customTools`에 '작업 체이닝' 도구 추가.

---

### 2.3 즐겨찾기 화면 세트 (Quick View)

**시나리오**: "아침 점검 루틴 실행해줘"

사전 등록된 화면 세트를 순서대로 열어 보여줍니다.

```
아침 점검 루틴 예시:
1. 대시보드 전체 현황 확인 (3초 대기)
2. 순단장애 미마감 건 확인
3. 예지보전 신규 티켓 확인
4. 서버 모니터링 이상 여부 확인
5. "아침 점검 완료. 이상 N건 발견" 보고
```

**필요 개발**: 루틴(매크로) 등록/관리 UI. localStorage 기반 루틴 저장. Page-Agent `systemInstructions`에 루틴 목록 주입.

---

### 2.4 알람 상관 분석 안내

**시나리오**: "이 알람과 관련된 다른 알람이나 장애가 있어?"

```
동작 흐름:
1. 현재 보고 있는 알람의 장비명/IP 추출
2. 순단장애 화면에서 같은 장비 관련 건 검색
3. Syslog에서 동일 장비 이력 검색
4. 관련 건이 있으면 연관 정보 요약 표시
```

**필요 개발**: Page-Agent `customTools`에 '상관분석' 도구 추가. 장비/IP 기반 크로스 검색 로직.

## 4. 추가 개발 제안

### 4.1 `instructions.system` 설정 (우선순위: 높음)

Page-Agent 초기화 시 NIA 도메인 지식을 주입합니다. 현재 코드에는 이 설정이 없습니다.

```javascript
this.pageAgent = new PageAgent({
  model: 'gemini-2.5-flash',
  baseURL: '/gemini-api',
  apiKey: process.env.VUE_APP_GEMINI_API_KEY,  // 환경변수로 이동
  language: 'ko-KR',
  instructions: {
    system: `당신은 NIA 네트워크 관제 플랫폼의 AI 어시스턴트입니다.
      사용자는 네트워크 운용자이며, 알람 관리, 장애 대응, 성능 모니터링을 수행합니다.

      화면 이동 시 좌측 사이드바 메뉴를 클릭하여 이동하세요.
      데이터 조회 시 필터 조건을 먼저 설정한 후 '조회' 버튼을 클릭하세요.

      [중요] 날짜 선택이 필요할 때는 반드시 set_date 도구를 사용하세요.
      el-date-picker를 직접 클릭하지 마세요. DOM 클릭으로는 날짜 변경이 불가합니다.

      주요 용어:
      - 순단장애: 짧은 시간 동안 발생하는 네트워크 단절 장애
      - 예지보전: 광모듈의 장애를 사전에 예측하는 기능
      - SOP: 표준운영절차 (Standard Operating Procedure)
      - 인지: 알람을 확인했음을 표시하는 처리
      - 마감: 장애 처리를 완료하는 행위
      - TT: Trouble Ticket (장애 티켓)`,
  },
})
```

### 4.2 `instructions.getPageInstructions` 콜백 (우선순위: 높음)

화면별 맞춤 지침을 제공하여 Page-Agent의 정확도를 높입니다.

```javascript
instructions: {
  // ...system은 위와 동일
  getPageInstructions: (url) => {
    // Vue Router의 hash mode를 고려하여 path 추출
    const hash = window.location.hash
    const path = hash.replace('#', '').split('?')[0]
    const pageInstructions = {
      '/dashBoard/index': `
        이 화면은 메인 대시보드입니다.
        상단에 AI제어 시스템 처리량 차트가 있습니다.
        중앙에 IP-SDN 네트워크와 전송망 알람 테이블이 있습니다.
        알람 테이블에서 행을 클릭하면 상세 정보를 볼 수 있습니다.
        '인지' 버튼으로 알람을 인지 처리할 수 있습니다.
        날짜 변경은 set_date 도구를 사용하세요.`,
      '/transientOutage/transientOutageList': `
        이 화면은 순단장애 목록입니다.
        각 티켓 카드에 상태(INIT/ACK/FIN), 케이블 정보, 영향 건수가 표시됩니다.
        '인지' 버튼으로 확인, '마감' 버튼으로 종료 처리합니다.
```

# 날짜 문제 해결 Plan

## 문제 정의

PageAgent는 텍스트 기반 DOM 추출 방식(simplified HTML)으로 페이지의 인터랙티브 요소를 파악합니다. `el-date-picker`(Element UI)는 클릭 시 캘린더 팝업이 `document.body` 아래에 동적으로 생성되고, 날짜 셀을 클릭해야 값이 설정되는 복합 컴포넌트입니다.

**실패 원인 분석**:

```
[PageAgent DOM 추출]
   ↓
페이지의 인터랙티브 요소를 번호로 인덱싱
   ↓
[0]<input class="el-date-editor" placeholder="Start date" />  ← 이것만 보임
   ↓
LLM: "clickElement(0)" 실행 → 캘린더 팝업 열림
   ↓
[DOM 다시 추출]
   ↓
[43]<td class="available">1</td>    ← 날짜 셀 수십개가 한꺼번에 등장
[44]<td class="available">2</td>
[45]<td class="available">3</td>
... (28~31개 셀 + 이전/다음 월 버튼 + 년/월 네비게이션)
   ↓
LLM: 어느 셀이 원하는 날짜인지 혼동 → 실패
```

1. 캘린더 팝업이 body 아래에 동적 생성되어, 최초 DOM 추출 시점에 존재하지 않음
2. 팝업이 열린 후에도 날짜 셀이 수십 개여서 LLM이 올바른 셀을 식별하기 어려움
3. 현재 월이 아닌 다른 월/년을 선택하려면 이전/다음 월 화살표까지 조작해야 하는 다단계 인터랙션
4. daterange 타입은 시작일과 종료일 두 번의 선택이 필요

### 방안 A: Custom Tool — `set_date` 도구 추가 (1순위)

page-agent의 `customTools` API를 활용하여, 날짜 설정 전용 도구를 LLM에게 제공합니다. LLM이 날짜 변경이 필요할 때 DOM 클릭 대신 이 도구를 호출합니다.

**원리**: Vue 2의 `__vue__` 인스턴스에 접근하여 v-model을 JavaScript로 직접 변경

```javascript
import { PageAgent, tool } from 'page-agent'
import { z } from 'zod/v4'

customTools: {
  set_date: tool({
    description: `날짜 입력 필드의 값을 설정합니다. el-date-picker 컴포넌트의 날짜를 변경할 때 반드시 이 도구를 사용하세요.
      clickElement로 날짜 선택기를 조작하지 마세요 - 동작하지 않습니다.
      selector: 날짜 필드를 식별하는 CSS 선택자 또는 placeholder 텍스트
      date: YYYY-MM-DD 형식의 날짜
      endDate: (범위 선택 시) YYYY-MM-DD 형식의 종료일`,
    inputSchema: z.object({
      selector: z.string().describe(
        'CSS selector (e.g. ".el-date-editor") or placeholder text (e.g. "Start date") to find the date picker'
      ),
      date: z.string().describe('Start date in YYYY-MM-DD format (e.g. "2026-03-01")'),
      endDate: z.string().optional().describe(
        'End date in YYYY-MM-DD format for range pickers (e.g. "2026-03-31")'
      ),
    }),
    execute: async function (input) {
      const { selector, date, endDate } = input

      // 입력 검증
      if (!/^\d{4}-\d{2}-\d{2}$/.test(date)) {
        return '오류: date는 YYYY-MM-DD 형식이어야 합니다 (예: 2026-03-01)'
      }
      if (endDate && !/^\d{4}-\d{2}-\d{2}$/.test(endDate)) {
        return '오류: endDate는 YYYY-MM-DD 형식이어야 합니다'
      }

      const escapedSelector = selector.replace(/'/g, "\\'")
      const script = `
        (function() {
          // 1단계: 날짜 필드 찾기
          let el = document.querySelector('${escapedSelector}');

          if (!el) {
            // placeholder 텍스트로 탐색
            const inputs = document.querySelectorAll('.el-date-editor input, .el-date-editor .el-range-input');
            for (const inp of inputs) {
              if (inp.placeholder && inp.placeholder.includes('${escapedSelector}')) {
                el = inp.closest('.el-date-editor');
                break;
              }
            }
          }

          if (!el) {
            // n번째 date picker로 탐색 (예: "1", "2")
            const idx = parseInt('${escapedSelector}');
            if (!isNaN(idx)) {
              const pickers = document.querySelectorAll('.el-date-editor');
              el = pickers[idx] || null;
            }
          }

          if (!el) return 'ERROR: 날짜 필드를 찾을 수 없습니다. selector: ${escapedSelector}';

          // 2단계: Vue 인스턴스 접근
          let vueInst = el.__vue__;
          if (!vueInst) {
            // 부모 요소에서 찾기
            let parent = el.parentElement;
            while (parent && !parent.__vue__) parent = parent.parentElement;
            vueInst = parent ? parent.__vue__ : null;
          }
          if (!vueInst) return 'ERROR: Vue 인스턴스를 찾을 수 없습니다';

          // 3단계: 날짜 설정
          const startDate = new Date('${date}T00:00:00');
          if (isNaN(startDate.getTime())) return 'ERROR: 유효하지 않은 날짜: ${date}';

          // el-date-picker의 type 확인
          const pickerType = vueInst.type || vueInst.$attrs?.type || '';
          const isRange = pickerType.includes('range');

          if (isRange) {
            ${endDate
              ? `const end = new Date('${endDate}T23:59:59');
                 if (isNaN(end.getTime())) return 'ERROR: 유효하지 않은 종료일: ${endDate}';
                 vueInst.$emit('input', [startDate, end]);
                 return 'SUCCESS: 날짜 범위 설정 완료 (${date} ~ ${endDate})';`
              : `vueInst.$emit('input', [startDate, startDate]);
                 return 'SUCCESS: 날짜 범위 설정 완료 (${date} ~ ${date})';`
            }
          } else {
            vueInst.$emit('input', startDate);
            return 'SUCCESS: 날짜 설정 완료 (${date})';
          }
        })()
      `
      const result = await this.pageController.executeJavascript(script)
      if (result.message && result.message.startsWith('SUCCESS')) {
        return result.message.replace('SUCCESS: ', '')
      }
      return result.message || '날짜 설정에 실패했습니다.'
    },
  }),
},
```

**장점**:
- 기존 Vue 컴포넌트 수정 불필요
- LLM이 description을 읽고 자연스럽게 도구를 선택
- 모든 el-date-picker에 범용 적용 (date, daterange, datetimerange)
- selector로 CSS 선택자, placeholder 텍스트, 인덱스 번호 모두 지원

**단점**:
- Vue 2의 `__vue__` 인스턴스 접근에 의존 (Vue 3에서는 `__vue_app__` 등으로 변경됨)
- `$emit('input', ...)` 호출이 Vuex/parent의 watch를 트리거하지 않을 수 있음 → 필요 시 `$set` 사용

**구현 단계**:
1. `chatbot/index.vue`의 PageAgent 초기화에 customTools.set_date 추가
2. instructions.system에 "날짜 변경 시 반드시 set_date 도구 사용" 지침 추가
3. 대시보드의 단일 날짜 picker → CompInquiryPannel의 daterange picker 순서로 테스트
4. 실패 시 `$emit` 대신 Vue 인스턴스의 data를 직접 수정하는 방식으로 폴백

### 방안 C: 날짜 컴포넌트 래퍼 (3순위)

`el-date-picker`를 감싸는 래퍼 컴포넌트를 만들어, PageAgent가 인식할 수 있는 숨겨진 text input을 함께 렌더링합니다.

```vue
<!-- CompDatePickerAgent.vue -->
<template>
  <div class="date-picker-agent-wrapper">
    <el-date-picker v-bind="$attrs" v-model="internalValue" v-on="$listeners" />
    <!-- PageAgent용 숨겨진 입력 필드 -->
    <input
      class="agent-date-input"
      type="text"
      :placeholder="`날짜 직접 입력 (${formatHint})`"
      :value="formattedValue"
      @change="onAgentInput"
      style="width: 1px; height: 1px; opacity: 0.01; position: absolute;"
    />
  </div>
</template>

<script>
export default {
  name: 'CompDatePickerAgent',
  inheritAttrs: false,
  props: { value: [Date, Array, String] },
  computed: {
    formatHint() {
      return this.$attrs.type?.includes('range') ? 'YYYY-MM-DD ~ YYYY-MM-DD' : 'YYYY-MM-DD'
    },
    formattedValue() {
      if (!this.value) return ''
      if (Array.isArray(this.value)) {
        return this.value.map(d => this.formatDate(d)).join(' ~ ')
      }
      return this.formatDate(this.value)
    },
    internalValue: {
      get() { return this.value },
      set(val) { this.$emit('input', val) },
    },
  },
  methods: {
    formatDate(d) {
      if (!d) return ''
      const date = new Date(d)
      return `${date.getFullYear()}-${String(date.getMonth()+1).padStart(2,'0')}-${String(date.getDate()).padStart(2,'0')}`
    },
    onAgentInput(e) {
      const text = e.target.value.trim()
      if (text.includes('~')) {
        const [start, end] = text.split('~').map(s => new Date(s.trim()))
        if (!isNaN(start) && !isNaN(end)) this.$emit('input', [start, end])
      } else {
        const date = new Date(text)
        if (!isNaN(date)) this.$emit('input', date)
      }
    },
  },
}
</script>
```

**장점**: PageAgent가 일반 input처럼 `inputText`로 날짜를 타이핑할 수 있음
**단점**:
- 모든 el-date-picker 사용처를 래퍼로 교체해야 함 (20+ 파일)

### 방안 E: transformPageContent로 날짜 필드 메타 주입 (보조)

DOM 추출 결과에 현재 날짜 필드 값과 인덱스 정보를 추가합니다.

```javascript
transformPageContent: async (content) => {
  const datePickers = document.querySelectorAll('.el-date-editor')
  let hint = ''
  datePickers.forEach((dp, i) => {
    const inputs = dp.querySelectorAll('input')
    const values = Array.from(inputs).map(inp => inp.value).filter(Boolean).join(' ~ ')
    const placeholder = inputs[0]?.placeholder || ''
    hint += `\n[DatePicker #${i}] placeholder="${placeholder}" currentValue="${values}" → set_date 도구로 변경 가능`
  })
  return content + (hint ? '\n--- 날짜 필드 현황 ---' + hint : '')
},
```
