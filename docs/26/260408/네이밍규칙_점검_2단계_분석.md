## 4. API sqlId 네이밍 불일치

### 4.1 PMM 도메인인데 MBA 접두사 사용

예지보전(PMM) 화면에서 사용하는 API인데 `MBA_` 접두사가 붙어 있음:

| 파일 | 현재 sqlId | 권장 sqlId |
|------|-----------|-----------|
| `ModalPredictiveReviewOpinion.vue` | `SELECT_MBA_PREDICTIVE_REVIEW` | `SELECT_PMM_PREDICTIVE_REVIEW` |
| `ModalPredictiveReviewOpinion.vue` | `UPDATE_MBA_PREDICTIVE_REVIEW` | `UPDATE_PMM_PREDICTIVE_REVIEW` |
| `ModalPredictiveReviewOpinion.vue` | `DELETE_MBA_PREDICTIVE_REVIEW` | `DELETE_PMM_PREDICTIVE_REVIEW` |
| `PredictiveMaintenanceTicket.vue` | `SELECT_MBA_REPEATER_SLOT_DATA` | 공용 API — 변경 검토 필요 |

> **주의**: API sqlId 변경 시 백엔드(MyBatis mapper XML)도 동시 수정 필요

### 4.2 SELECT 패턴 불일치

MBA(순단장애)와 PMM(예지보전)의 SELECT 패턴이 다름:

| 도메인 | 패턴 | 예시 |
|--------|------|------|
| MBA | `SELECT_MBA_*` | `SELECT_MBA_TICKET_CUR_LIST` |
| PMM | `SELECT_TICKET_PMM_*` 또는 `SELECT_PMM_*` | `SELECT_TICKET_PMM_LIST`, `SELECT_PMM_TOPOLOGY_LIST` |

**권장 통일 패턴**: `SELECT_{도메인}_{대상}` (예: `SELECT_MBA_TICKET_LIST`, `SELECT_PMM_TICKET_LIST`)
