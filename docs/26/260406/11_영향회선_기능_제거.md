# 영향회선 기능 전면 제거

## 변경 이유

영향회선 기능은 엔진 측 데이터(테이블, 매핑, 노드명)가 부재하여 구현 불가로 확인됨.
기능 자체를 빼는 것으로 결론.

## 변경 파일

| 파일 | 변경 |
|------|------|
| `TransientOutageList.vue` | el-tabs 구조 → 일반 그리드, 영향회선 탭/data/computed/method 제거 |
| `PredictiveMaintenanceTicket.vue` | 영향회선 pane 제거, 토폴로지 전체 확장, splitpanes 제거 |
| `transmissionFaultAnalytics.xml` | `SELECT_MBA_PMM_INFLUENCECIRCUIT_LIST` 쿼리 삭제 |
| `transmissionFaultAnalytics.java` | `SELECT_MBA_PMM_INFLUENCECIRCUIT_LIST` 매퍼 인터페이스 삭제 |

## 순단장애 (TransientOutageList.vue)

### template 변경

| Before | After |
|--------|-------|
| `el-tabs` (알람 리스트 탭 + 영향회선 리스트 탭) | 일반 div (알람 리스트 그리드만) |
| 영향서비스 카운트 표시 | 제거 |

### script 제거

| 항목 | 내용 |
|------|------|
| `data.influencecircuitList` | 영향회선 데이터 배열 |
| `computed.influenceGridTable` | 영향회선 그리드 설정 (컬럼 12개) |
| `methods.loadInfluencecircuitList()` | 영향회선 API 호출 |
| `methods.loadList()` 내 호출 | `loadInfluencecircuitList` 호출 제거 |
| `methods.exportExcel()` 내 분기 | 영향회선 제목 분기 제거 |

## 예지보전 (PredictiveMaintenanceTicket.vue)

### template 변경

| Before | After |
|--------|-------|
| splitpanes (토폴로지 pane + 영향회선 pane) | 토폴로지만 전체 높이 확장 |
| 영향회선 el-tabs | 제거 |

### script 제거

| 항목 | 내용 |
|------|------|
| `import Splitpanes, Pane` | 불필요 — 제거 |
| `components: { Splitpanes, Pane }` | 제거 |
| `data.influencecircuitList` | 제거 |
| `computed.influenceGridTable` | 제거 (컬럼 12개) |
| `methods.loadInfluencecircuitList()` | 제거 |
| `methods.loadList()` | 빈 콜백으로 변경 |
| `methods.exportExcel()` 내 분기 | 제거 |

## 백엔드

| 항목 | 내용 |
|------|------|
| `SELECT_MBA_PMM_INFLUENCECIRCUIT_LIST` (XML) | 쿼리 전체 삭제 (WHERE 1=0 + 주석) |
| `SELECT_MBA_PMM_INFLUENCECIRCUIT_LIST` (Java) | 매퍼 인터페이스 삭제 |
