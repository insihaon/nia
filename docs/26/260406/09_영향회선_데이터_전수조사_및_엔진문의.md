안녕하세요,
순단장애/예지보전 화면의 "영향회선 리스트" 기능 관련하여 확인 요청드립니다.

### 현황

순단장애/예지보전 화면에 "영향회선 리스트" 탭이 있으며, 해당 장애 구간을 경유하는 서비스 회선 목록을 보여주는 기능입니다.
현재 이 기능은 `mba.tb_ticket_ic_mst` 테이블을 참조하도록 되어 있으나, DB 전체 스키마를 조사한 결과 **해당 테이블이 존재하지 않아** 항상 빈 리스트가 표시되고 있습니다.
추가로 NIA DB의 전체 스키마(mba, nia, rca, resource, event 등)를 전수 조사했으나, 아래의 데이터가 모두 부재합니다:

1. **전표↔서비스 매핑**: 특정 장애(TICKET_ID)가 어떤 서비스 회선에 영향을 미치는지의 연결 정보
2. **장비↔서비스 경유 정보**: ROADM 장비를 경유하는 서비스 회선 목록
3. **노드명 매핑**: 전표의 한글 장비명(서울NIA)과 서비스 마스터의 영문 약어(SEOUL) 간 매핑

### 확인 요청

1. **영향회선 데이터 생성 계획**
   - 향후 MBA 엔진에서 순단장애/예지보전 전표 생성 시, 해당 구간의 영향회선 정보를 함께 산출하여 DB에 적재하는 기능의 개발 계획이 있는지요?

2. **필요 테이블 사양 협의**
   - UI 측에서는 아래 정보가 필요합니다. 테이블 구조 협의가 가능한지요?
     - 전표 ID (TICKET_ID)
     - 서비스/회선 식별자 및 명칭
     - 고객명
     - A/Z측 위치 정보

3. **대안 여부**
   - 혹시 영향회선 정보를 산출할 수 있는 다른 데이터 소스나 외부 시스템 연동 방안이 있는지요?

현재 UI 코드(탭, 그리드, API 호출)는 모두 준비되어 있으므로, 데이터가 준비되면 쿼리 수정만으로 즉시 연동 가능합니다.
