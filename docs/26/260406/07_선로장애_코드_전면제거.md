# 선로장애(CableCut) 코드 전면 제거

## 변경 이유

순단장애(TransientOutage) 화면은 MBA 전용이며, 선로장애(CableCut) 데이터는 표시되지 않음.
원래 RCA 공통 모달이었던 코드를 순단장애 전용으로 단순화.

## 변경 파일

| 파일 | 변경 |
|------|------|
| `ModalRcaProcessHandling.vue` | isCable/isMba 분기 제거, 선로장애 전용 UI/로직 제거 |
| `ModalRcaBatchProcessing.vue` | causesite/brokencause/responsibility 주석 및 컬럼 제거 |
| `transmissionFaultAnalytics.xml` | 선로장애 쿼리 5개 삭제, LEVEL 컬럼 제거, RCA 분기 제거 |
| `transmissionFaultAnalytics.java` | 매퍼 인터페이스 5개 삭제 |
| `nia.js` | apiUserProcess 시그니처 변경 |

---

## 1. ModalRcaProcessHandling.vue

### 제거된 data

| 항목 | 용도 |
|------|------|
| `isMba`, `isCable` | 페이지 유형 분기 플래그 |
| `rootDisorderList` | 광케이블 리스트 데이터 |
| `faultreason`, `faultcharge` | 원인분류/시설분류 옵션 |
| `reason`, `charge`, `clear` | 분류 선택값 |
| `reasonOp`, `chargeOp`, `clearOp` | select disabled 상태 |
| `fault_type` (userInput) | 선로장애 fault_type |

### 제거된 computed

| 항목 | 용도 |
|------|------|
| `classificationForm` | 원인분류/시설분류/마감코드 el-select 폼 |
| `radioForm` | 선로장애 일치여부 / MBA 장애유형 라디오 |
| `rootDisorderGridData` | 광케이블 리스트 그리드 |
| `columns` | 미사용 빈 computed |

### 제거된 methods

| 항목 | 용도 |
|------|------|
| `setPageType()` | isMba/isCable 분기 |
| `isDisabledClassification()` | 분류 select disabled 판정 |
| `onChangeClassification()` | 분류 선택 변경 핸들러 |
| `loadFaultList()` | REASON/CHARGE 코드 API 조회 |
| `onChangeSelectedVal()` | forceUpdate 트리거 |
| `handleInspectorProcess()` | inspector 전용 (미사용) |

### 제거된 template

| 영역 | 용도 |
|------|------|
| `v-if="!isMba"` 광케이블 리스트 그리드 | 선로장애 전용 |
| `v-if="!isMba && !isCable"` AI 분석 결과 | 기타 유형 전용 |
| `v-if="..." 장애 상세 코드` | 선로/기타 전용 |
| `v-if="..." TOPAS 분류 선택 (classificationForm)` | 선로장애 전용 |
| `'cable'` CSS 클래스 조건 | 선로장애 레이아웃 |

### 단순화된 로직

| 항목 | Before | After |
|------|--------|-------|
| `inputForm` ticketIdProp | 3분기 (isCable/isMba/기타) | `ticket_id` 고정 |
| `getProcessParam` TICKET_ID | `rca_ticketid ?? inspector_seq` | `ticket_id` |
| `getProcessParam` TICKET_TYPE | `isCable ? 'CIT2' : 'RT'` | `ticket.ticket_type` |
| `getProcessParam` LEVEL 할당 | isCable 분기 | 제거 |
| `loadSavedData` | 3분기 + IS_MBA 조건부 | `IS_MBA: true` 고정 |
| `getTitle` | CIT2 → '선로장애' prefix | prefix 제거 |
| 소속 본부 라벨 | `소속 본부` | `소속 팀` (정정) |

## 3. transmissionFaultAnalytics.xml

### 삭제된 쿼리 (5개)

| sqlId | 용도 |
|-------|------|
| `SELECT_FAULTREASON_LV1_LIST` | 고장원인 1단계 코드 조회 |
| `SELECT_FAULTREASON_LV2_LIST` | 고장원인 2단계 코드 조회 |
| `SELECT_FAULTREASON_LV3_LIST` | 고장원인 3단계 코드 조회 |
| `SELECT_FAULTCHARGE_LV1_LIST` | 시설분류 1단계 코드 조회 |
| `SELECT_FAULTCHARGE_LV2_LIST` | 시설분류 2단계 코드 조회 |

### SELECT_TICKET_ALARM_BATCH_LIST

| 제거 | 내용 |
|------|------|
| `H.REASON_LEVEL1 AS BROKENCAUSE` | 고장원인 alias |
| `H.CHARGE_LEVEL1 AS RESPONSIBILITY` | 책임분류 alias |

### SELECT_TICKET_HANDLING_CURRENT_LIST 단순화

| Before | After |
|--------|-------|
| IS_MBA 분기 (MBA / RCA 테이블) | MBA 테이블 고정 |
| REASON_LEVEL1~3, CHARGE_LEVEL1~2, FIN_LEVEL1~3 | 제거 |
| TICKET_TYPE, MAX_DAYS 조건 | 제거 (TICKET_ID만) |

### USER_PROCESS_MBA

| 제거 | 내용 |
|------|------|
| `<if test="REASON_LEVEL1 != null">` INSERT/VALUES/UPSERT | 원인분류 조건부 저장 |
| `<if test="CHARGE_LEVEL1 != null">` INSERT/VALUES/UPSERT | 시설분류 조건부 저장 |

## 5. nia.js — apiUserProcess 시그니처 변경

| Before | After |
|--------|-------|
| `apiUserProcess(params, 'mba')` | `apiUserProcess('USER_PROCESS_MBA', params)` |
| `sqlId: USER_PROCESS_${type.toUpperCase()}` | `sqlId: sqlId` (직접 전달) |

`apiRcaRequest(sqlId, params)`와 동일한 시그니처로 통일하여 sqlId 검색 가능.
