# 순단장애/예지보전 데이터 점검 및 엔진팀 문의

## 1. MBA 엔진 파이프라인 분석

`EngineNiaOpticalPm` 소스 코드를 추적한 결과, 엔진 코드 자체는 **실시간 수집 구조**입니다.

### 전체 파이프라인

```
[15분 스케줄러] LinkageSchdulerService (@Scheduled cron = "0 0/15 * * * *")
    ↓
EmsMmcHdlService.pmCollectHdl()
    ↓
EMS 장비 접속 (TL1 소켓, 61.252.53.34)
    → RTRV-PM 명령으로 ROADM 광 성능 데이터 수집
    ↓
parsingHdlService → DB 저장 (mba.tb_roadm_optical_performance_hist)
    ↓
pmMapper.callPmPrepro() → 전처리 프로시저
    ↓
MbaAiModelPrdAmqp → RabbitMQ 큐로 AI 모델에 전송
    ↓
[AI 모델 분석] (외부 — 이 소스코드에 없음)
    ↓
nia.MbaAiModelResultIndexDirectly 큐로 결과 수신
    ↓
MbaAiResultListener (prepro-mba)
    ↓
LineMonitoringHdlService
    → valueMax - valueMin >= 3 이면 순단 판정
    ↓
TicketDataService → TicketCreateService
    → mba.tb_mba_ticket_current INSERT (ticketType="PF", rootCauseType="MomentaryBreakoff")
    → RabbitMQ로 UI에 TICKET_NEW 이벤트 전송
```

### 핵심 발견

- 엔진 코드 구조는 **실시간** (15분 주기 EMS 장비 직접 수집)
- `fault_time`은 `lowPmDataDtoList.get(0).getOcrtime()`에서 가져옴 → EMS에서 수집한 PM 데이터의 시간
- **fault_time이 전부 2024-08-14인 이유**: 코드 문제가 아니라, 아래 중 하나로 추정
  - (A) EMS 장비가 과거 데이터를 반복 제공
  - (B) AI 모델이 테스트 데이터로 고정 응답
  - (C) linkage 모듈(EMS 수집)이 실제로는 중단된 상태

## 2. 데이터 비교 분석

### RCA PF ≠ MBA 순단장애

| 구분 | ticket_type | root_cause_type | 건수 | 의미 |
|------|------------|-----------------|------|------|
| RCA PF | `PF` | `Performance` | 59,504건 | 성능 장애 (RCA 엔진) |
| MBA 순단장애 | `PF` | `MomentaryBreakoff` | 116건 | 순단장애 (MBA 엔진) |

ticket_type이 같은 `PF`이지만 **완전히 다른 시스템, 다른 분석 결과**.
- RCA PF: RCA 엔진(EngineNiaServer2021)이 생성 → 실시간 가동 중
- MBA PF: MBA 엔진(EngineNiaOpticalPm)이 생성 → 시뮬레이션 데이터만 존재

### 순단장애 (mba.tb_mba_ticket_current)

| 항목 | 값 |
|------|-----|
| 전체 건수 | 116건 |
| 최근 생성 | 2026-03-26 |
| 오늘 건수 | 0건 |
| fault_time | 전부 `2024-08-14 02:30:00` |
| trunk_name | 전부 `서울NIA--수원성균관대` |

### 예지보전 (mba.tb_roadm_predictive_maintenance_ticket)

| 항목 | 값 |
|------|-----|
| 전체 건수 | 7건 |
| 최근 생성 | 2026-03-25 |
| 내용 | 전부 "광레벨 장기 변화 예측 경보" |

## 3. 결론

엔진 코드(`EngineNiaOpticalPm`)는 실시간 수집 구조이지만, 현재 DB에 적재되는 데이터는 과거 데이터(2024-08-14)만 반복되고 있음.

원인은 엔진 코드가 아니라 **파이프라인의 외부 요소**(EMS 장비 또는 AI 모델)에 있을 가능성이 높음.

- **fault_time(ocrtime)이 전부 `2024-08-14 02:30:00`으로 동일**

코드 구조는 실시간인데, 적재되는 데이터가 동일한 과거 시간(2024-08-14)만 반복되는 것으로 보아, 아래 중 하나로 추정하고 있습니다.

### 확인 요청
