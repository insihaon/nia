# 순단장애 / 예지보전 — 팝업(모달) 분석

## 1. 순단장애 (TransientOutage)

| 모달 파일 | 활성화 트리거 | 설명 |
|-----------|-------------|------|
| `ModalRcaProcessHandling.vue` | 티켓의 **인지** 또는 **마감** 버튼 클릭 | `onOpenModalProcess('ack', row)` 또는 `onOpenModalProcess('fin', row)` 호출 → 해당 티켓의 A/Z 시설 목록 로드 후 모달 open |
| `ModalRcaBatchProcessing.vue` | 티켓의 **TT일괄처리** 버튼 클릭 | `onOpenModalProcess('batch', row)` 호출 → 동일 trunk_name 기준 전표 조회 후 모달 open |
| `ModalRcaBatchFix.vue` | TT일괄처리 모달 내 **조치** 버튼 클릭 | `ModalRcaBatchProcessing` 내부에서 전표 선택 후 `openFixModal()` 호출 → 하위 모달 open |

### 호출 흐름

```
TransientOutageList.vue
  ├─ [인지] 버튼 → onOpenModalProcess('ack') → ModalRcaProcessHandling.vue
  ├─ [마감] 버튼 → onOpenModalProcess('fin') → ModalRcaProcessHandling.vue
  └─ [TT일괄처리] 버튼 → onOpenModalProcess('batch') → ModalRcaBatchProcessing.vue
                                                          └─ [조치] 버튼 → ModalRcaBatchFix.vue
```

## 2. 예지보전 (PredictiveMaintenance)

| 모달 파일 | 활성화 트리거 | 설명 |
|-----------|-------------|------|
| `ModalPredictiveReviewOpinion.vue` | 티켓의 **검토 의견** 버튼 클릭 | `handleReviewOpinion(row)` 호출 → 해당 티켓의 검토 의견 목록 모달 open |

### 호출 흐름

```
PredictiveMaintenanceTicket.vue
  └─ [검토 의견] 버튼 → handleReviewOpinion(ticket) → ModalPredictiveReviewOpinion.vue
```
