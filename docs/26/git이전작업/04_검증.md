# 04. 검증

> 작성일: 2026-04-08

---

## 1. 검증 목적

신규 레포 (`insihaon/nia`)에 **모든 이력이 정확히 이전**되었는지 확인한다.

---

## 2. 검증 절차

### Step 1: 신규 레포 클론 (별도 디렉토리)

```bash
# 별도 디렉토리에 full clone
cd D:\01_source_code
git clone https://github.com/insihaon/nia.git nia_verification
cd nia_verification
```

### Step 2: main 브랜치 이력 비교

```bash
# 커밋 수 비교
git log --oneline main | wc -l
# 기대값: 1,122

# 최신 커밋 해시 비교
git log --oneline -1 main
# 기대값: 95ac71ea (또는 push 이후 추가 커밋이 있으면 그 해시)

# 최초 커밋 확인
git log --oneline --reverse main | head -1
# 기대값: ec6ed5c7
```

### Step 3: Archive 브랜치 이력 비교

```bash
# 모든 archive 브랜치 가져오기
git fetch origin 'refs/heads/archive/*:refs/remotes/origin/archive/*'

# 각 브랜치 커밋 수 확인
echo "=== web2024 ===" && git log --oneline origin/archive/web2024-original | wc -l
echo "=== engine2021 ===" && git log --oneline origin/archive/engine2021-original | wc -l
echo "=== optical-pm ===" && git log --oneline origin/archive/optical-pm-original | wc -l
echo "=== ip-sdn ===" && git log --oneline origin/archive/ip-sdn-original | wc -l
echo "=== legacy-2021 ===" && git log --oneline origin/archive/legacy-2021-original | wc -l
echo "=== chatbot-tool ===" && git log --oneline origin/archive/chatbot-tool-original | wc -l
echo "=== topology2d ===" && git log --oneline origin/archive/topology2d-original | wc -l
```

**기대값:**

| 브랜치 | 기대 커밋 수 | 확인 |
|--------|-------------|------|
| web2024-original | 781 | [ ] |
| engine2021-original | 221 | [ ] |
| optical-pm-original | 4 | [ ] |
| ip-sdn-original | 3 | [ ] |
| legacy-2021-original | 1,023 | [ ] |
| chatbot-tool-original | 5 | [ ] |
| topology2d-original | 2 | [ ] |

### Step 4: 파일 무결성 확인

```bash
# main 브랜치의 tree 해시 비교
# 원본 레포에서:
cd "D:\01_source_code\0. nia_source"
git rev-parse main^{tree}

# 신규 레포에서:
cd "D:\01_source_code\nia_verification"
git rev-parse main^{tree}

# 두 해시가 동일하면 파일 내용이 100% 동일
```

### Step 5: subtree merge 이력 연결 확인

```bash
# subtree add 커밋의 부모가 원본 이력과 연결되어 있는지 확인
git log --oneline --graph 99be41f3 -5
# EngineNiaServer2021의 subtree add 커밋 → 두 부모가 보여야 함

git log --oneline --graph 03cef6b9 -5
# EngineNiaOpticalPm의 subtree add 커밋
```

## 4. 검증 실패 시 대응

| 실패 항목 | 원인 | 대응 |
|----------|------|------|
| 커밋 수 불일치 | shallow push 또는 일부 object 누락 | `--mirror` 재시도 |
| archive 브랜치 없음 | push 누락 | `git push nia archive/*` |
| tree 해시 불일치 | push 중 파일 변경 | 원본에서 재push |
| subtree 연결 끊김 | object 누락 | bundle에서 복원 후 재push |
