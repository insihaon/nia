# 06. 비상 복구 플랜

> 작성일: 2026-04-08

---

## 1. 목적

이전 작업 중 또는 이후에 문제가 발생했을 때의 복구 절차를 정의한다.

---

## 2. 복구 시나리오

### 시나리오 A: Push 실패 — 원본은 무사

**증상**: 신규 레포에 push가 완료되지 않음
**영향**: 없음 (원본 레포는 변경되지 않음)
**복구**:

```bash
# 원인 파악 후 재시도
git push --mirror nia

# 또는 개별 push
git push nia main
git push nia --all
```

---

### 시나리오 B: 신규 레포에 이력 누락 발견

**증상**: 검증 시 커밋 수 불일치 또는 archive 브랜치 누락
**영향**: 신규 레포 이력 불완전
**복구**:

```bash
# 누락된 브랜치만 재push
git push nia archive/web2024-original

# 또는 전체 재push
git push nia --all --force
```

---

### 시나리오 C: 로컬 .git 손상

**증상**: git 명령어 오류, object 접근 불가
**영향**: 로컬 작업 불가
**복구**:

```bash
# Bundle에서 복원
cd D:\01_source_code
git clone ../NIA_TOTAL_PROJECT_backup_20260408.bundle nia_restored
cd nia_restored

# remote 재설정
git remote add origin https://github.com/insihaon/nia.git
```

---

### 시나리오 D: 신규 레포 접근 불가 (이전 완료 후)

**증상**: 신규 레포가 삭제되거나 접근 불가
**영향**: push/pull 불가
**복구**:

```bash
# 로컬에 전체 이력이 있으므로, 새 레포를 만들어 재push
# 또는 기존 레포로 복원
git remote set-url origin git@github.com:dloere/NIA_TOTAL_PROJECT.git
```

---

### 시나리오 E: 대용량 파일로 push 거부

**증상**: `GH001: Large files detected. File X is 150.00 MB`
**영향**: push 불가
**복구**:

```bash
# 1. 대용량 파일 식별
git rev-list --objects --all | \
  git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | \
  awk '/^blob/ && $3 > 104857600 {print $3, $4}' | \
  sort -rnk1

# 2. Git LFS로 전환
git lfs install
git lfs track "*.jar"       # 예시: jar 파일이 큰 경우
git lfs track "*.min.js"    # 예시: 번들 파일이 큰 경우

# 3. 이력에서 대용량 파일 처리
# git filter-repo 사용 (주의: 커밋 해시 변경됨)
pip install git-filter-repo
git filter-repo --strip-blobs-bigger-than 100M

# 또는 BFG Repo Cleaner 사용
java -jar bfg.jar --strip-blobs-bigger-than 100M
```

> 이 시나리오는 커밋 해시가 변경되므로 archive 브랜치도 재생성 필요

---

## 3. 연락처 및 에스컬레이션

| 상황 | 대응 |
|------|------|
| GitHub 서비스 장애 | https://githubstatus.com 확인 |
| 레포 권한 문제 | GitHub Organization 관리자에게 연락 |
| 데이터 손실 의심 | bundle 백업에서 즉시 복원 |

---

## 4. 복구 우선순위

1. **최우선**: 로컬 레포 무결성 보존 (bundle 백업)
2. **높음**: 기존 레포 접근 유지 (안정화 기간 동안)
3. **중간**: 신규 레포 이력 완전성
4. **낮음**: archive 브랜치 완전성 (재생성 가능)
