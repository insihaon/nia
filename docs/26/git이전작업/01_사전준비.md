## 2. 사전 준비 명령어

### 2.1 백업 생성

```bash
# 작업 디렉토리: D:\01_source_code\0. nia_source

# 방법 1: git bundle (권장 — 전체 이력 포함)
git bundle create ../NIA_TOTAL_PROJECT_backup_20260408.bundle --all

# 방법 2: .git 폴더 직접 복사
# 별도 드라이브로 .git/ 전체를 복사
```

### 2.2 신규 레포 접근 테스트

```bash
# HTTPS 접근 테스트
git ls-remote https://github.com/insihaon/nia

# SSH 접근 테스트 (SSH 키 설정 시)
git ls-remote git@github.com:insihaon/nia.git
```

### 2.3 대용량 파일 탐색

```bash
# 100MB 이상 파일 탐색
git rev-list --objects --all | \
  git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | \
  sed -n 's/^blob //p' | \
  sort -rnk2 | \
  head -20
```

> 100MB 이상 파일이 있으면 GitHub push가 거부됨 → Git LFS 전환 필요

## 3. 위험 관리

| 위험 | 영향 | 대응 |
|------|------|------|
| push 중 네트워크 끊김 | 부분 전송 | bundle 백업으로 재시도 |
| 대용량 파일 push 거부 | push 실패 | Git LFS로 전환 후 재시도 |
| 인증 실패 | push 불가 | PAT(Personal Access Token) 생성 |
| 신규 레포에 기존 커밋 존재 | 충돌 | `--force` 필요 (빈 레포 권장) |
