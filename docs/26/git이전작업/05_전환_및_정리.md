## 1. 목적

검증 완료 후 로컬 작업 환경을 신규 레포 기준으로 전환하고,
기존 레포 참조를 정리한다.

---

## 2. 로컬 remote 전환

### 2.1 remote 변경

```bash
cd "D:\01_source_code\0. nia_source"

# 기존 remote 이름 변경 (백업 용도)
git remote rename origin old-origin

# 신규 remote를 origin으로 설정
git remote rename nia origin

# 또는 한 번에:
# git remote set-url origin https://github.com/insihaon/nia.git

# 확인
git remote -v
# origin  https://github.com/insihaon/nia.git (fetch)
# origin  https://github.com/insihaon/nia.git (push)
# old-origin  git@github.com:dloere/NIA_TOTAL_PROJECT.git (fetch)
# old-origin  git@github.com:dloere/NIA_TOTAL_PROJECT.git (push)
```

### 2.2 upstream 추적 설정

```bash
# main 브랜치가 신규 origin을 추적하도록 설정
git branch --set-upstream-to=origin/main main

# 확인
git branch -vv
# * main  95ac71ea [origin/main] ...
```

## 3. 안정화 기간

### 3.1 병행 운영 (권장: 1-2주)

- 신규 레포(`origin`)로 push/pull 작업
- 기존 레포(`old-origin`)는 읽기 전용으로 유지
- 문제 발생 시 `old-origin`으로 즉시 복원 가능

### 3.2 병행 기간 중 확인 사항

- [ ] 신규 레포에서 push/pull 정상 동작
- [ ] 다른 팀원이 있다면 clone/fetch 정상 동작
- [ ] GitHub Actions/CI가 있다면 정상 동작
- [ ] 기존 이력 조회 (`git log`, `git blame`) 정상

## 4. 최종 정리

### 4.1 기존 remote 제거 (안정화 완료 후)

```bash
# 안정화 기간 종료 후, 기존 remote 제거
git remote remove old-origin
```

### 4.2 기존 레포 처리

| 옵션 | 설명 |
|------|------|
| Archive | GitHub에서 Archive 설정 → 읽기 전용 |
| 삭제 | 완전 삭제 (비권장 — 최소 6개월 유지 권장) |
| 리다이렉트 | README에 신규 레포 URL 안내 |

**권장**: Archive 처리 + README에 이전 안내 메시지 추가

```markdown
# NIA_TOTAL_PROJECT

> **이 레포지토리는 이전되었습니다.**
> 
> 신규 레포: https://github.com/insihaon/nia
> 이전일: 2026-04-XX
```

### 4.3 bundle 백업 보관

- bundle 파일은 최소 6개월 보관
- 신규 레포 안정화 확인 후 삭제 가능
