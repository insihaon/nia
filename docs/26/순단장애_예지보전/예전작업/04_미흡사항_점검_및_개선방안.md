### 엔진 수정 필요

#### PM-E1. 토폴로지 맵에서 PM 전용 SQL 대신 MBA SQL 사용

```
파일: PredictiveMaintenance/component/Map2D.vue:158~159
```

```javascript
// PM 화면인데 MBA 토폴로지 SQL을 호출
const result2 = await apiRcaRequest('SELECT_MBA_TOPOLOGY_LIST', { TRUNK_NAME: ... })
```

`SELECT_MBA_TOPOLOGY_LIST`는 3개 컬럼만 반환: `TRUNK_NAME, SYSNAME, ROUTENUM`

PM 토폴로지에는 `RX, TX, SPAN_GAIN, NODE_TOTAL_DEVIATION` 등이 필요하다.
이 값들은 `SELECT_PMM_TOPOLOGY_LIST`에 있지만 Map2D 컴포넌트가 이것을 호출하지 않음.

**증상**: PM 토폴로지 맵에서 노드 정보 표시 옵션(RX/TX, Span, NTD)이 모두 빈 값
**필요**: Map2D 컴포넌트가 PM 컨텍스트일 때 `SELECT_PMM_TOPOLOGY_LIST`를 사용하도록 분기 처리. 이것은 UI 수정이지만, `SELECT_PMM_TOPOLOGY_LIST`가 Map2D가 기대하는 노드 형식과 일치하는지 검증 필요.

#### PM-E2. SELECT_PMM_TOPOLOGY_LIST에서 TIME_STAMP 파라미터 무시

```
파일: PredictiveMaintenanceTicket.vue:245

파일: transmissionFaultAnalytics.xml:640~783
```

**필요**: SQL에 `WHERE ... AND TIME_STAMP = #{TIME_STAMP}` 조건 추가 또는 파라미터 제거

### 기능별 영향도

| 기능 | 현재 상태 | 차단 원인 |
|------|----------|----------|
| 전표 목록 조회 | **동작** | - |
| 전표 상세 보기 | **부분 동작** | Z측 시스템명 누락, 장비유형 NULL |
| 토폴로지 맵 (순단) | **부분 동작** | 반복기 슬롯 필터 불가 |

| 마감(FIN) 처리 | **미동작** | 원인/시설분류 드롭다운 빈 목록 |
| TT 일괄처리 | **미동작** | 배치 경보 목록 빈 결과 |

| 검토의견 등록 | **동작** | 일부 UX 문제 있음 |
| 검토의견 수정/삭제 | **동작** | - |
| PMM 차트 연결 | **미동작** | device_name → sysname 오류 |
