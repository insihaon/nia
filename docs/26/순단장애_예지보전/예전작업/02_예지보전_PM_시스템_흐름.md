# 예지보전(PM) 시스템 흐름

> 예지보전 = PM (Predictive Maintenance)
> NTD(Node Total Deviation) 추이 분석 + AI 예측으로 광모듈 장애를 사전 감지하여 전표를 생성하고, 운용자가 검토 의견을 등록하는 시스템

---

## 1. 전체 흐름 요약

```
[엔진] 일일 광성능 집계 → NTD 변화량 판정 → 전표 생성 → [UI] 전표 목록 조회
                                                              ↓
                                                      토폴로지 맵 + 영향 회선 확인
                                                              ↓
                                                      운용자 검토의견 등록
                                                              ↓
                                                     [BE] DB 저장 (PostgreSQL)
```

---

## 2. 엔진 흐름 (Engine)

프로젝트: `apps/EngineNiaOpticalPm/`

### Step 1: 일일 광성능 집계 (매일 01:05)

```
파일: predictiveMaintenance-domain/src/.../service/PmDailyService.java
함수: performanceDailyHdl()

[원본 수집 데이터]
  NIA.TB_PERFORMACE_MST
    ↓
일별 집계 계산:
  - NTD (Node Total Deviation): 노드 전체 편차
  - Span Loss: 구간 손실
  - Node Gain: 노드 이득
  - Reference 기준값
    ↓
[일별 집계 테이블]
  MBA.TB_ROADM_OPTICAL_PERFORMANCE_DAILY
```

### Step 2: 전표 생성 (매일 02:05)

```
파일: predictiveMaintenance-domain/src/.../service/ticket/PmTicketService.java
함수: ticketHdlProcessor()
  │
  ├─ Case 1: NTD 변화량 기반
  │    파일: .../service/ticket/TicketCaseOneService.java
  │    함수: ticketCaseOneHdlProcessor()
  │    판정: |NTD_VARIANCE| > MAX(2.0dB, 3σ)
  │    사유: "Node Total Deviation 변화"
  │
  ├─ Case 2: 추가 판정 로직
  │    파일: .../service/ticket/TicketCaseTwoService.java
  │    함수: ticketCaseTwoHdlProcessor()
  │
  └─ AI 경고일 기반
       파일: .../service/ticket/PmWarningDateService.java
       함수: warningDateCheck()
       참조: MBA.TB_PM_AI_PREDICT_RESULT
       사유: "광레벨 장기 변화 예측 경보"

    ↓
[전표 저장]
  MBA.TB_ROADM_PREDICTIVE_MAINTENANCE_TICKET
    - ticketno: 전표 번호
    - issue_date: 발행일
    - trunk_name: 캐리어명
    - direction: UP/DOWN
    - reason: 판정 사유
    - target_sysname: 영향 노드 (콤마 구분)
    - node_total_deviation: NTD 값
```

## 6. 순단장애와의 차이

| 항목 | 순단장애(MBA) | 예지보전(PM) |
|------|-------------|-------------|
| **감지** | 실시간 광레벨 급변 (3dB 이상) | NTD 추이 + AI 예측 |
| **전표 생성** | 즉시 (RabbitMQ 비동기) | 스케줄러 (매일 02:05) |
| **UI 갱신** | WebSocket 실시간 | API 폴링 (수동 새로고침) |
| **처리 방식** | 인지(ACK) → 마감(FIN) | 검토의견 등록/수정/삭제 |
| **전표 테이블** | `TB_MBA_TICKET_CURRENT` | `TB_ROADM_PREDICTIVE_MAINTENANCE_TICKET` |
| **메인 Vue** | `TransientOutageList.vue` | `PredictiveMaintenanceTicket.vue` |
| **처리 SQL** | `USER_PROCESS_MBA` | `UPDATE/DELETE_MBA_PREDICTIVE_REVIEW` |
| **엔진 핵심** | `TicketCreateService` | `PmTicketService` + `TicketCaseOneService` |
| **경보음** | 순단장애.mp3 | 없음 |
