# 순단장애 전표 테이블 불일치 분석

> 분석일: 2026-04-01
> 분석 요청: SELECT_TICKET_CUR_LIST 쿼리가 MBA.TB_MBA_TICKET_CURRENT 테이블을 사용해야 하는지 검증

---

## 1. 문제 요약

순단장애(MBA) 전표의 **저장(Write)** 과 **조회(Read)** 가 서로 다른 테이블을 참조하고 있다.

| 동작 | 엔진 | 테이블 | 스키마 |
|------|------|--------|--------|
| **전표 생성 (Write)** | EngineNiaOpticalPm (신규 엔진) | `MBA.TB_MBA_TICKET_CURRENT` | MBA |
| **전표 조회 (Read)** | Web2024 UI (SELECT_TICKET_CUR_LIST) | `RCA.TB_RCA_TICKET_CURRENT` | RCA |

**핵심 질문**: SELECT_TICKET_CUR_LIST를 `MBA.TB_MBA_TICKET_CURRENT`로 변경하면 되는가?

---

## 2. MBA.TB_MBA_TICKET_CURRENT가 순단장애 테이블인 근거

### 2.1 코드 흐름 추적

신규 엔진(`EngineNiaOpticalPm`)의 순단장애 전표 생성 흐름:

```
MbaTicketMsgListener.onMessage()           ← RabbitMQ 수신 (engineMbaTicketQueue)
  └─ TicketCreateService.createMbaTicket()
     ├─ createTicketEntity()               ← MbaTicketCurEntity 빌드
     │   ├─ ticketType("PF")              → Performance Fault
     │   ├─ rootCauseType("MomentaryBreakoff")  → ★ 순단장애 식별자
     │   ├─ ticketRcaResultCode("OPTICAL_CABLE_LOW_SIGNAL")
     │   ├─ ticketRcaResultDtlCode("광레벨 순단 저하")
     │   ├─ rootCauseDomain("ROADM")
     │   └─ status("INIT")
     │
     ├─ ticketRepository.save(ticketEntity)     → MBA.TB_MBA_TICKET_CURRENT 저장
     ├─ ticketCableRepository.saveAll(...)      → MBA.TB_MBA_TICKET_CABLE 저장
     ├─ lowOptPerfRepository.saveAll(...)       → MBA.TB_ROADM_LOW_OPTICAL_PERFORMANCE 저장
     └─ engineToUiTicketPrdAmqp.sendMessageCmd() → WebSocket UI 알림 (TICKET_NEW)
```

### 2.2 순단장애 식별자 매칭 확인

| 항목 | 신규 엔진 (Write) | UI 쿼리 (Read) | 일치 여부 |
|------|-------------------|----------------|:---------:|
| ROOT_CAUSE_TYPE | `"MomentaryBreakoff"` | `WHERE T.ROOT_CAUSE_TYPE = 'MomentaryBreakoff'` (IS_MBA 필터) | **O** |
| TICKET_TYPE | `"PF"` | - | - |
| RCA_RESULT_CODE | `"OPTICAL_CABLE_LOW_SIGNAL"` | 조회 컬럼에 포함 | **O** |
| ROOT_CAUSE_DOMAIN | `"ROADM"` | 조회 컬럼에 포함 | **O** |

### 2.4 결론

`MBA.TB_MBA_TICKET_CURRENT`는 **순단장애 전용 전표 테이블이 확실하다**.
- `rootCauseType("MomentaryBreakoff")`은 UI 쿼리의 `IS_MBA` 필터와 정확히 동일
- 데이터 흐름: 저광레벨 감지 → RabbitMQ → `TicketCreateService` → `MBA.TB_MBA_TICKET_CURRENT` 저장 → WebSocket → UI

## 5. JOIN 의존성 분석

SELECT_TICKET_CUR_LIST는 다음 RCA 스키마 테이블들을 JOIN한다:

| JOIN 테이블 | MBA 스키마 대응 테이블 | 상태 |
|------------|:--------------------:|------|
| `rca.tb_rca_ticket_al_current` | **없음** | MBA 스키마에 알람 라인 테이블 미존재 |
| `rca.tb_rca_ticket_handling_status_current` | **없음** | MBA 스키마에 처리 상태 테이블 미존재 |
| `rca.tb_rca_ticket_cnt` | **없음** | MBA 스키마에 카운트 테이블 미존재 |

### JOIN별 역할과 영향

**1) `rca.tb_rca_ticket_al_current` (알람 라인)**
- 역할: 전표의 Root Cause 장비명(SYSNAMEA/Z), 포트(PORTA/Z), 장비타입 표시
- UI 표시 항목: `ROOT_CAUSE_SYSNAMEA`, `ROOT_CAUSE_PORTA`, `ROOT_CAUSE_EQUIP_TYPEA/Z`, `MBA_SYSNAMEA/Z`
- MBA 대응: `MBA.TB_MBA_TICKET_CABLE` (sysnamea/z, ptpnamea/z 컬럼 보유)

**2) `rca.tb_rca_ticket_handling_status_current` (처리 상태)**
- 역할: 전표 접수/완료 처리 정보 (처리자, 처리일시, 처리내용)
- UI 표시 항목: `HANDLING_CONTENT`, `HANDLING_USER`, `HANDLING_TIME`, `HANDLING_FIN_CONTENT`, `RCA_ACCURACY`
- MBA 대응: **없음** — 순단장애 처리 상태를 별도로 관리하는 MBA 테이블이 없음

**3) `rca.tb_rca_ticket_cnt` (카운트)**
- 역할: 관련 알람 수, 자식 전표 수
- UI 표시 항목: `TOTAL_ALARM_COUNT`, `TOTAL_RELATED_ALARM_COUNT`, `CHILD_COUNT`
- MBA 대응: **없음** — MBA.TB_MBA_TICKET_CURRENT의 `TOTAL_RELATED_ALARM_CNT` 컬럼으로 일부 대체 가능

```sql
    SELECT T.*
    FROM MBA.TB_MBA_TICKET_CURRENT T    -- ★ 변경: rca.tb_rca_ticket_current → MBA.TB_MBA_TICKET_CURRENT
    WHERE (
        T.PARENT_TICKET_ID IS NULL
        -- IS_MBA 필터 불필요: MBA 테이블 자체가 순단장애 전용

)

    , T.PARENT_TICKET_ID                                            AS PARENT_TICKET_ID

    , T.ROOT_CAUSE_CODE                                             AS ROOT_CAUSE_CODE
    , T.STATUS                                                      AS STATUS
    -- , T.TICKET_FM_NONFM                                          AS TICKET_FM_NONFM          -- [MBA 미존재] RCA 전용 컬럼

    , T.TICKET_RCA_RESULT_DTL_CODE                                  AS TICKET_RCA_RESULT_DTL_CODE

    -- , TO_CHAR(STATUS.HANDLING_TIME, ...) AS HANDLING_TIME                                       -- [MBA 미존재]
    , NULL                                                          AS HANDLING_TIME
    -- , STATUS.HANDLING_USER                                        AS HANDLING_USER              -- [MBA 미존재]
    , NULL                                                          AS HANDLING_USER
    -- , STATUS.HANDLING_DEPT                                        AS HANDLING_DEPT              -- [MBA 미존재]
    , NULL                                                          AS HANDLING_DEPT
    -- , STATUS.HANDLING_AGENCY                                      AS HANDLING_AGENCY            -- [MBA 미존재]
    , NULL                                                          AS HANDLING_AGENCY
    -- , TO_CHAR(STATUS.HANDLING_ACK_TIME, ...) AS HANDLING_ACK_TIME                               -- [MBA 미존재]
    , NULL                                                          AS HANDLING_ACK_TIME
    -- , STATUS.HANDLING_FIN_USER                                    AS HANDLING_FIN_USER          -- [MBA 미존재]
    , NULL                                                          AS HANDLING_FIN_USER
    -- , STATUS.HANDLING_FIN_DEPT                                    AS HANDLING_FIN_DEPT          -- [MBA 미존재]
    , NULL                                                          AS HANDLING_FIN_DEPT
    -- , STATUS.HANDLING_FIN_AGENCY                                  AS HANDLING_FIN_AGENCY        -- [MBA 미존재]
    , NULL                                                          AS HANDLING_FIN_AGENCY

    -- ====================================================================
    -- [IS_NEW_TICKET 판정] — 그대로 사용 가능
    -- ====================================================================
    , (SELECT CURRENT_TIMESTAMP - T.TICKET_GENERATION_TIME <= '5 minute') AS IS_NEW_TICKET

    -- ====================================================================
    -- [CNT JOIN 대체] rca.tb_rca_ticket_cnt
    -- MBA에 카운트 테이블 없음 → TOTAL_RELATED_ALARM_CNT로 부분 대체
    -- ====================================================================
    , T.TOTAL_RELATED_ALARM_CNT                                     AS TOTAL_ALARM_COUNT         -- ★ 대체: MBA 컬럼
    -- , AL2.ROOT_CAUSE_CHECKA                                       AS ROOT_CAUSE_CHECKA         -- [MBA 미존재]
    , 0                                                             AS ROOT_CAUSE_CHECKA         -- ★ 대체: 기본값
    -- , AL2.ROOT_CAUSE_CHECKZ                                       AS ROOT_CAUSE_CHECKZ         -- [MBA 미존재]
    , 0                                                             AS ROOT_CAUSE_CHECKZ         -- ★ 대체: 기본값
    , T.TOTAL_RELATED_ALARM_CNT                                     AS TOTAL_RELATED_ALARM_COUNT -- ★ 대체: MBA 컬럼
    , 0                                                             AS CHILD_COUNT               -- ★ 대체: MBA 전표는 자식 전표 없음

FROM sub_ticket T
    -- ====================================================================
    -- [JOIN 변경] rca.tb_rca_ticket_al_current → MBA.TB_MBA_TICKET_CABLE
    -- 원본: 복잡한 ROW_NUMBER + EQUIP_TYPE_ORDER_NUM 서브쿼리
    -- MBA: CABLE 테이블의 첫 번째 레코드를 가져옴 (IS_ROOT_ROW_SIGNAL_LINE = true 우선)
    -- ====================================================================
    LEFT OUTER JOIN (
        SELECT *
            , ROW_NUMBER() OVER (PARTITION BY TICKET_ID ORDER BY TICKET_CABLE_ID) AS ORDER_NUM
        FROM MBA.TB_MBA_TICKET_CABLE
    ) CABLE ON T.TICKET_ID = CABLE.TICKET_ID AND CABLE.ORDER_NUM = 1

    -- ====================================================================
    -- [JOIN 제거] rca.tb_rca_ticket_handling_status_current
    -- MBA 스키마에 대응 테이블 없음 → SELECT에서 NULL로 대체
    -- TODO: MBA 처리 상태 테이블 생성 후 이 JOIN 추가
    -- ====================================================================

    -- ====================================================================
    -- [JOIN 제거] rca.tb_rca_ticket_cnt
    -- MBA 스키마에 대응 테이블 없음 → TOTAL_RELATED_ALARM_CNT로 대체
    -- ====================================================================

ORDER BY T.TICKET_GENERATION_TIME DESC, T.TICKET_ID DESC
;
```

## 8. 결론

### 8.1 단순 테이블명 변경으로는 불가능

`SELECT_TICKET_CUR_LIST`에서 `rca.tb_rca_ticket_current`를 `MBA.TB_MBA_TICKET_CURRENT`로 단순 교체하면 **다음 문제 발생**:

1. **컬럼 누락**: `TICKET_UPDATE_TIME`, `ROW_SIGNAL_LINE_CNT` 등 MBA 테이블에 없는 컬럼 참조로 SQL 오류
2. **JOIN 실패**: `rca.tb_rca_ticket_al_current`, `rca.tb_rca_ticket_handling_status_current`, `rca.tb_rca_ticket_cnt`가 MBA 스키마에 대응 테이블이 없으므로 JOIN 불가
3. **처리 워크플로 단절**: 접수(ACK)/완료(FIN) 처리 정보를 저장할 테이블이 MBA 스키마에 없음

### 8.2 변경 영향 범위

순단장애 UI에서 사용하는 **총 12개 쿼리** 중:
- **3개 필수 변경**: `SELECT_TICKET_CUR_LIST`, `SELECT_TICKET_ROOT_ALARM_LIST`, `SELECT_TICKET_HANDLING_CURRENT_LIST`
- **9개 변경 불필요**: 이미 MBA 스키마 사용 또는 비활성(`WHERE 1=0`) 상태

### 8.3 핵심 선행 작업: MBA 처리 상태 테이블

쿼리를 MBA로 전환해도 **전표 접수/완료 기능이 동작하지 않는다**. MBA 스키마에 `tb_rca_ticket_handling_status_current`에 대응하는 처리 상태 관리 기능이 없기 때문이다.

**선행 작업 우선순위:**

| 순번 | 작업 | 필요성 |
|:----:|------|:------:|
| 1 | MBA 처리 상태 테이블 설계 및 생성 | **필수** |
| 2 | `SELECT_TICKET_CUR_LIST` MBA 버전 적용 | **필수** |
| 3 | `SELECT_TICKET_ROOT_ALARM_LIST` MBA 버전 적용 | **필수** |
| 4 | `SELECT_TICKET_HANDLING_CURRENT_LIST` MBA 버전 적용 | **필수** |
| 5 | `MB.TB_MBA_TICKET_CURRENT`에 `TICKET_UPDATE_TIME` 컬럼 추가 | 권장 |
| 6 | 전표 접수/완료 API (MERGE_HANDLE_STATUS) MBA 버전 작성 | **필수** |
