# 실시간 모니터링 훅 연동 (최종)

## 목적

Claude Code 작업 이벤트를 외부 대시보드(agents-observe)로 실시간 전송하여, 에이전트의 활동을 모니터링한다.

## 연동 구조 (v2 - 전역 통합)

```
Claude Code (훅 이벤트 발생)
  ├─ 전역 훅 (~/.claude/settings.json)
  │    └─ observe_cli.mjs → HTTP POST → agents-observe 대시보드
  │
  └─ 프로젝트 훅 (.claude/settings.json)
       └─ handle-*.sh → moai hook (MoAI 내부 로직)
```

**관심사 분리**: observe(모니터링)는 전역에서, moai(워크플로우)는 프로젝트에서 각각 독립 관리.

## 아키텍처

### 전역 모니터링 (observe_cli.mjs)

```
~/.claude/hooks/observe_cli.mjs
  │
  ├─ 프로젝트 이름 결정
  │   우선순위: --project > env > observe_projects.json > CLAUDE_PROJECT_DIR
  │
  ├─ 이벤트 타입 분류
  │   hook → tool_use, prompt, agent_lifecycle, task, session, team, notification, context
  │
  ├─ 메타데이터 추출 (훅별 풍부한 데이터)
  │
  ├─ 세션 통계 누적 (tool 호출 수, prompt 수, agent 수, 에러 수)
  │
  ├─ 에이전트 duration 추적 (SubagentStart/Stop 간 경과시간)
  │
  ├─ 토큰 근사치 계산 (content length / 3.5)
  │
  └─ HTTP POST → localhost:4981/api/events
```

### 프로젝트 이름 매핑

`~/.claude/hooks/observe_projects.json`:
```json
{
  "0. nia_source": "NIA"
}
```

디렉토리명 → 사람이 읽기 좋은 프로젝트명으로 매핑. 새 프로젝트 추가 시 이 파일만 수정.

## 전역 훅 등록 (8개 이벤트)

`~/.claude/settings.json`:

| 이벤트 | 타입 분류 | 전송 데이터 |
|--------|----------|------------|
| UserPromptSubmit | `prompt` | promptLength, promptPreview, estimatedTokens |
| PostToolUse | `tool_use` | toolName, filePath, command, editSize, outputLength, estimatedTokens, subagentType |
| PostToolUseFailure | `tool_use` | toolName, error, isInterrupt |
| SubagentStart | `agent_lifecycle` | agentType, agentName (+ 시작시간 기록) |
| SubagentStop | `agent_lifecycle` | agentType, agentName, durationSec, lastMessagePreview, estimatedTokens |
| Stop | `session` | lastMessagePreview, sessionStats (누적 통계) |
| TaskCompleted | `task` | taskId, taskSummary, agentName |
| Notification | `notification` | notificationType, message |

## 대시보드에서 볼 수 있는 정보

### 프로젝트 수준
- 프로젝트별 이벤트 필터링 (NIA, MyApp 등)
- 세션별 활동 타임라인

### 도구 활동 (tool_use)
- 어떤 도구가 어떤 파일을 수정했는지
- Bash 명령어 실행 내역 (300자까지)
- Edit/Write 크기 (변경 전/후)
- 출력 크기 및 토큰 근사치

### 에이전트 활동 (agent_lifecycle)
- 어떤 에이전트가 생성/종료되었는지
- 에이전트 실행 시간 (durationSec)
- 에이전트의 마지막 응답 미리보기
- 에이전트가 사용한 토큰 근사치

### 세션 통계 (session)
- 세션 종료 시 누적 통계:
  - 총 도구 호출 수
  - 프롬프트 수
  - 에이전트 생성 수
  - 에러 발생 수
  - 세션 시작 시간

### 팀 활동 (team)
- 팀원 idle 상태 및 작업 요약
- 태스크 완료 이벤트

## 파일 구조

```
~/.claude/
  hooks/
    observe_cli.mjs          ← 전역 모니터링 스크립트 (유일한 관리 지점)
    observe_projects.json    ← 프로젝트 이름 매핑
  settings.json              ← 전역 훅 등록 (8개 이벤트)

.claude/hooks/moai/
  handle-user-prompt-submit.sh  ← moai hook 전용 (observe 없음)
  handle-post-tool.sh           ← moai hook 전용 (observe 없음)
  handle-stop.sh                ← moai hook 전용 (observe 없음)
  handle-task-completed.sh      ← moai hook 전용 (observe 없음)

%TEMP%/observe-agents/
  session_*.json              ← 세션별 누적 통계
  *_*.json                    ← 에이전트 시작시간 추적 (자동 정리)
```

## 새 프로젝트 추가 방법

1. `~/.claude/hooks/observe_projects.json`에 매핑 추가:
   ```json
   {
     "0. nia_source": "NIA",
     "my-new-project": "NewProject"
   }
   ```
2. 끝. 전역 훅이 자동으로 새 프로젝트를 인식함.

## 제한사항

- **토큰 사용량**: Claude Code 훅에서 실제 토큰 수를 제공하지 않음. `estimatedTokens`는 `문자수 / 3.5`로 계산한 근사치.
- **세션 통계**: 프로세스 간 공유이므로 temp 파일 기반. 비정상 종료 시 통계가 남을 수 있음.
- **에이전트 duration**: SubagentStart와 SubagentStop 간 시간차로 계산. 훅 실행 지연 포함.
