# 챗봇 모듈 사용법

## 📁 모듈 구조

```
chatbot/1.chatbot/
├── config.py                 # 설정 관리
├── elasticsearch_client.py   # ES 클라이언트
├── data_loader.py            # 데이터 로드
├── search_engine.py          # 검색 엔진
├── data_initializer.py       # 데이터 초기화 (최초 한 번만)
├── chatbot_service.py        # 챗봇 서비스
└── main.py                   # 메인 실행 로직
```

## 🔧 각 모듈의 역할

### `config.py`
- Elasticsearch 연결 설정
- 텍스트 분석기 설정
- 데이터 매핑 설정

### `elasticsearch_client.py`
- ES 연결 및 테스트
- 인덱스 생성/관리
- 기본 CRUD 작업

### `data_loader.py`
- CSV 데이터 파싱
- Elasticsearch 대량 색인
- 데이터 검증

### `search_engine.py`
- 텍스트 검색 기능
- 검색 결과 포맷팅
- 카테고리별 검색

### `data_initializer.py`
- **최초 한 번만 실행**
- CSV 데이터 로드 및 색인
- 초기 설정 완료 후 종료

### `chatbot_service.py`
- 실제 챗봇 서비스
- 검색 기능 제공
- 사용자 질문 처리

### `main.py`
- 챗봇 서비스 실행
- 기능 테스트 및 시연

## 📋 사용 예시

### 데이터 초기화
```bash
python data_initializer.py
# 결과: CSV 데이터를 ES에 색인 완료
```

### 챗봇 서비스 실행
```bash
python main.py
# 결과: 대화형 검색 인터페이스 시작
```

### 샘플 테스트 실행
```bash
python sampletest.py
# 결과: 검색 기능 테스트 및 시연
```

## ⚠️ 주의사항

1. **데이터 초기화는 최초 한 번만 실행**
2. **Elasticsearch가 실행 중이어야 함**
3. **모듈 간 의존성 순서 준수**

## 🎯 핵심 특징

- **모듈화**: 기능별로 명확하게 분리
- **재사용성**: 각 모듈을 독립적으로 사용 가능
- **유지보수성**: 버그 수정 시 해당 모듈만 수정
- **확장성**: 새로운 기능 추가 시 관련 모듈만 수정
