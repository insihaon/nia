# 리눅스 서버에서 Elasticsearch 데이터 입력 가이드

## 📋 개요
이 가이드는 리눅스 서버에서 명령어를 통해 Elasticsearch에 단일 데이터를 입력하는 방법을 설명합니다.

## 🚀 사용 방법

### 1. 기본 사용법 (권장)
```bash
# 스크립트 실행 권한 부여
chmod +x insert_single_data.sh

# 스크립트 실행
./insert_single_data.sh
```

### 2. 간단한 버전 사용법
```bash
# 스크립트 실행 권한 부여
chmod +x insert_simple.sh

# 기본 설정으로 실행
./insert_simple.sh

# 환경 변수로 설정 변경하여 실행
ES_URL="http://your-server:9200" ES_USER="admin" ES_PASS="password" ./insert_simple.sh
```

### 3. 직접 curl 명령어 사용
```bash
curl -X POST -u "elastic:changeme123" \
    "http://localhost:9200/chatbot-qa/_doc/1" \
    -H "Content-Type: application/json" \
    -d '{
        "question": "현재 발생한 장애를 보여줘",
        "answer": "http://116.89.191.47:8080/fault/current?status=active",
        "category": "장애 조회",
        "api_call": "/fault/current?status=active",
        "description": "현재 활성 장애 조회",
        "created": "'$(date -u +"%Y-%m-%dT%H:%M:%S.%3NZ")'"
    }'
```

## ⚙️ 환경 변수 설정

스크립트 실행 전 환경 변수를 설정할 수 있습니다:

```bash
# Elasticsearch URL
export ES_URL="http://your-server:9200"

# 사용자 인증 정보
export ES_USER="elastic"
export ES_PASS="your-password"

# 인덱스 이름
export INDEX_NAME="chatbot-qa"
```

## 🔍 데이터 확인

### 입력된 데이터 조회
```bash
curl -u "elastic:changeme123" "http://localhost:9200/chatbot-qa/_doc/1"
```

### 전체 인덱스 검색
```bash
curl -u "elastic:changeme123" "http://localhost:9200/chatbot-qa/_search"
```

### 인덱스 정보 확인
```bash
curl -u "elastic:changeme123" "http://localhost:9200/chatbot-qa"
```

## 🐳 Docker 환경에서 사용

Docker Compose로 Elasticsearch가 실행 중인 경우:

```bash
# 컨테이너 상태 확인
docker-compose ps

# Elasticsearch 로그 확인
docker-compose logs elasticsearch

# 스크립트 실행
./insert_single_data.sh
```

## ❗ 문제 해결

### 1. 연결 실패
- Elasticsearch가 실행 중인지 확인
- 포트 9200이 열려있는지 확인
- 방화벽 설정 확인

### 2. 인증 실패
- 사용자명과 비밀번호 확인
- Elasticsearch 보안 설정 확인

### 3. 권한 오류
```bash
# 스크립트 실행 권한 확인
ls -la insert_single_data.sh

# 권한 부여
chmod +x insert_single_data.sh
```

## 📊 입력되는 데이터 구조

첫 번째 줄 데이터:
- **질문**: "현재 발생한 장애를 보여줘"
- **답변**: "http://116.89.191.47:8080/fault/current?status=active"
- **카테고리**: "장애 조회"
- **API 호출**: "/fault/current?status=active"
- **설명**: "현재 활성 장애 조회"
- **생성일시**: 현재 UTC 시간
