### 5. data-flow.md (569줄)
**하이라이트**:
- 시스템 레벨 데이터 흐름 (수집 → 전처리 → 엔진 → 연계 → AI → 시각화)
- RabbitMQ 메시지 큐 토폴로지 (6개 계층):
  - 레벨 1: 수집 큐 (SNMP, Syslog, NetFlow, sFlow, ICMP)
  - 레벨 2: 처리 큐 (전처리 출력)
  - 레벨 3: 규칙 엔진 큐 (nia.engine.events)
  - 레벨 4: 연계 큐 (성능+토폴로지+알림 상관관계)
  - 레벨 5: 알림 큐 (Critical, Warning, Info)
  - 레벨 6: AI 분석 큐 (학습, 예측, 결과)

- HTTP 요청/응답 흐름 (실제 시나리오):
  - 성능 대시보드 조회 (GET /api/nia/performance)
  - 토폴로지 조회 (다중 소스 통합)

- 데이터베이스 저장 흐름:
  - 성능 메트릭 저장 및 파티셔닝
  - 알림 저장 및 히스토리 보관

- WebSocket 실시간 업데이트:
  - 브라우저 연결 수립
  - STOMP 구독
  - 메트릭 푸시 및 UI 렌더링

- 요청 생명주기 (엔드-투-엔드):
  - 총 90ms 엔드-투-엔드 지연 시간
  - T+0ms ~ T+90ms 타임라인

- 배치 데이터 처리:
  - AI 모델 야간 학습 (00:00 UTC ~ 03:30)
  - 데이터 수집, 정규화, 학습, 검증, 배포 프로세스

- 오류 처리 및 Dead Letter Queue (DLQ):
  - 처리 실패 메시지 관리
  - 재시도 및 모니터링

- 성능 최적화 전략:
  - 처리량: 초당 10,000+ 메시지
  - 캐싱 적중률: 70%+
  - 데이터베이스 쿼리 타임아웃: 30초

**대상 독자**: 아키텍트, 고급 개발자 (문제 추적, 성능 최적화)
